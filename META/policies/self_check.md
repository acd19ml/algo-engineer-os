# Self-Check Layer

> 自检题层规约。所有 `KNOWLEDGE/_self_check/<domain>.md` 必须遵守。

---

## 这一层是什么

自检题是 KNOWLEDGE 的**导航 + 测试层**，物理上独立于节点。

- KNOWLEDGE 节点：因果叙述形态的解释（"是什么、为什么、怎么推导"）
- 自检题：浅入深的提问序列（"你会不会、能不能复述、能不能跨节点综合"）

两层分离的原因：

1. **避免 spoiler**：自检题如果和答案/讲解同页，看题就看到答案，等于自检失效
2. **避免知识被问题绑架**：知识有自己的因果叙述线，问题只是测试层。"我不希望知识的存在是为了回答一个片面的问题"——节点的形态由内容决定，不由问题决定
3. **支持全局浅入深**：日常自检从上往下过一遍，浅题先答、深题后答；卡住就跳节点重学

---

## 文件组织

按 domain 一个 deck：

```
KNOWLEDGE/_self_check/
├── ml.md
├── nlp.md
├── optimization.md
├── pytorch.md
├── vision.md
├── methodology.md       # 学习方法 / 答题方法这类 meta-skill
└── transformer.md       # （新建域）
```

domain 边界和 `KNOWLEDGE/<domain>/` 一致。

跨 domain 的综合题（少数）放在源头 domain（题目主要触及哪个 domain）。

---

## 单条问题的格式

```
- [{难度}] 问题文本？ → `KNOWLEDGE/<domain>/<node>/`
- [{难度}] 问题文本？ → `KNOWLEDGE/<domain>/<node-a>/` + `KNOWLEDGE/<domain>/<node-b>/`
```

**难度三档**：

- `[浅]` —— 单一概念能不能复述；答出 1-2 句话就够
- `[中]` —— 设计动机 / 反事实 / 跨段推导；需要展开几句
- `[深]` —— 跨节点综合 / 边界情形 / Open Questions 类；需要把多个概念串起来

**链接规则**：

- 每题必须链接到至少一个节点（无节点可链 = 知识缺失，先建节点）
- 如果题在节点的某个具体段落，可以加 `#section-anchor`（GitHub 风格 anchor，按小标题转 lowercase-kebab）
- 跨节点综合题用 `+` 链多个节点

---

## 排序规则

deck 内部分两层排序：

1. **Topic 间顺序**：按学习的逻辑前后排（基础 → 派生）。比如 ml.md：表征学习 → 梯度流 → 初始化归一化 → 残差。
2. **Topic 内部顺序**：浅 → 中 → 深。

Deck 末尾可有一段 `## 跨节点综合`，放横跨多 topic 的深题。

---

## 题目来源（不再限定"卡住位置"）

旧规则曾要求自检题来源限定为对话 log 中"用户卡住 / 被纠正 / 被要求复述"的位置。**现在放宽**：

允许的来源：

- ✅ 对话 log 中卡住 / 被纠正的位置（仍然优质来源）
- ✅ 节点正文里被反事实推导驱动的关键判断（"如果不这么做会怎样"——可以拆成题）
- ✅ Open Questions 拆出的可回答部分
- ✅ 用户上传的面经 / 课程过往题（确认用户学过对应内容才能放进 self-check；没学过的进 `CAREER/interview-bank/` 标 `[未学]`）
- ✅ 跨节点综合题（LLM 判断节点之间的连接）

仍然禁止：

- ❌ LLM 拍脑袋编的"标准考点"，无对应节点支持
- ❌ 题目所触及的内容用户从未参与过（如果只是从 ppts/notes 里看的"标准应该问的题"）

---

## 答案规则

**答案不写在 deck 文件里**。卡住就点链接跳节点重学——这是设计本意。

例外：如果某条题非常短小、节点完全没必要为它建独立段落（比如一个数字回忆），可以在 deck 行下面缩进写 1 行答案，但用 `<details>` 折叠：

```
- [浅] BPE 的"加权频率"是指什么？ → `KNOWLEDGE/nlp/bpe-tokenization/`
  <details>pair 在某词内出现次数 × 该词词频，再对所有词求和</details>
```

少用。默认还是无答案 + 链接。

---

## Open Questions 类的题

Open Questions 拆出来的题特点：节点本身没给完整答案。这种题用 `[深]` 标签 + 在链接后加 `(open)`：

```
- [深] 梯度裁剪是否破坏方向信息？clip-by-norm vs clip-by-value 的选择依据？ → `KNOWLEDGE/ml/gradient-flow-deep-networks/#open-questions` (open)
```

这类题不要求"会答"，要求"知道这是个开问题，并能复述问题本身"。

---

## 触发时机

LLM 在以下时机更新 self-check decks（不是每次 triage 都改）：

- 新建节点：在对应 domain deck 加题
- 节点 README 大改（形态升级 / 内容扩展）：检查相关题是否需要改写或加深
- 用户在对话里展示出"对某概念已熟练"：考虑加入更深一档的题（但 LLM 只建议、用户决定，写进 Triage Report）
- 用户丢面经到 INBOX 且确认相关知识已学：可以从面经派生题进 self-check（仍然要符合质量规则）

## 写完前的自检清单

- [ ] 每题都有节点链接？
- [ ] 难度标签和题的实际深度对得上？
- [ ] Topic 顺序符合"浅 → 深"的学习逻辑？
- [ ] Topic 内部题目顺序也是"浅 → 深"？
- [ ] 答案没有泄漏到 deck 文件里（除少数 1 行 `<details>`）？
- [ ] 没有 LLM 拍脑袋的"标准考点"？
- [ ] Open Questions 类题标了 `(open)`？
