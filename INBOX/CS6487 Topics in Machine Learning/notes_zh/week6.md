## 概述 / 回顾

### 从手工设计特征到学习表示

传统图机器学习遵循两阶段流程：

1. **特征工程** — 基于数学概念（方差、稀疏性、平移、旋转、尺度等）手动设计节点级、边级和图级特征
2. **下游模型** — 训练 SVM 或神经网络，将手工设计的特征映射到标签

其局限在于：对于每个新任务和每个新图，特征工程都必须重新进行。

**特征表示方法的时间线**（如讲座幻灯片所示）：
- 1901年——早期统计方法
- 1936年——经典学习模型
- 1996–2005年——SVM、核方法
- 2012年——深度学习/可学习特征（如 AlexNet 时代）

### 传统图机器学习（回顾前几周）

```
输入图  -->  特征工程  -->  结构化学习算法  -->  预测
             （节点/边/图级别）
```

特征工程是瓶颈：它是手动的、特定任务的，并且需要领域专业知识。

## 表征学习

### 图表征学习：动机

**图表征学习**消除了手动设计特征的需要。与其使用固定特征，系统会*自动学习*从节点（或整个图）到低维向量空间的映射。

```
输入图  -->  [表征学习]  -->  特征嵌入  -->  下游任务
```

**目标**：高效的、*与任务无关*的图机器学习特征学习。

形式上，我们寻求映射：

```
f : u  -->  R^d
```

其中 `u` 是节点，`R^d` 是 $d$ 维嵌入空间。

### 为何使用嵌入？

嵌入允许：
- **相似性编码**：嵌入空间中的邻近反映网络中的相似性（例如，通过边连接的两个节点应该有相似的嵌入）。
- **多任务适用性**：单个嵌入可以被重用于许多下游任务，无需重新设计特征。

**节点嵌入（Node Embeddings）支持的下游任务（Downstream Tasks）：**
- 节点分类
- 链接预测
- 图分类
- 异常节点检测
- 聚类

**经典示例**：Zachary 空手道俱乐部网络中节点的2D嵌入（Perozzi 等，DeepWalk，KDD 2014）——来自同一社区的节点在嵌入空间中聚集在一起。

---

### 编码器-解码器框架（Encoder-Decoder Framework）

学习节点嵌入的通用框架由四个组件组成：

1. **编码器** — 将每个节点 `v` 映射到低维向量：
   ```
   ENC(v) = z_v  ∈ R^d
   ```

2. **节点相似度函数** — 定义两个节点在原始图中"相似"的含义（这是关键的设计选择）。

3. **解码器** — 将嵌入对映射到相似度分数：
   ```
   DEC(z_v, z_u) = z_v^T · z_u   （点积）
   ```

4. **优化目标** — 调整编码器参数，使得：
   ```
   similarity(u, v)  ≈  z_v^T · z_u
   ```
   即，嵌入的点积近似真实节点相似度。

#### 设置

给定图 `G = (V, E)`：
- `V` = 顶点集
- `A` = 邻接矩阵（假设为二值）
- 不假设额外的节点特征（嵌入纯粹从图结构中学习）

---

### "浅层"编码（Shallow Encoding / Embedding Lookup）

最简单的编码器是**嵌入查找表（Embedding Lookup Table）**：

```
ENC(v) = Z · v
```

其中：
- `Z ∈ R^{d × |V|}` — 嵌入矩阵（每节点一列）；这是我们**学习/优化**的对象
- `v ∈ {0,1}^|V|` — 节点 `v` 的独热指示向量（除节点 `v` 对应列为1外，其余全为0）

每个节点被分配一个唯一的、独立优化的嵌入向量。这种方法被 **DeepWalk** 和 **node2vec** 使用。

**需要优化的关键参数**：矩阵 `Z`，其中包含每个节点 `u` 的嵌入 `z_u`。

**注意**：浅层编码是**无监督/自监督**的：
- 不使用节点标签
- 不使用节点特征
- 目标：直接估计节点的坐标（嵌入），使得某些结构属性（由解码器捕获）得以保留
- 得到的嵌入是**与任务无关的**

---

### 如何定义节点相似度？

核心设计问题是：*什么时候两个节点应该有相似的嵌入？*

可能的定义：
- **直接连接** — 节点通过边相连
- **共享邻域** — 节点共享许多共同邻居
- **结构角色** — 节点在图中扮演相似的结构角色（例如，都是枢纽节点）

本讲座重点介绍**基于随机游走**的相似度。

## 节点嵌入的随机游走方法（Random Walk-based Methods for Node Embeddings）

### 符号说明

| 符号 | 含义 |
|--------|---------|
| `z_u` | 节点 `u` 的嵌入（我们优化的目标） |
| `P(v \| u)` | 模型预测的从 `u` 出发的随机游走中访问 `v` 的概率 |
| `N_R(u)` | 通过随机游走策略 `R` 得到的 `u` 的邻域 |
| `σ(z_i)` | Softmax：`exp(z_i) / Σ_j exp(z_j)` |
| `S(x)` | Sigmoid：`1 / (1 + exp(-x))` |

### 什么是随机游走（Random Walk）？

给定一个图和一个起始节点，在每一步：
1. 从邻居中均匀随机选择一个
2. 移动到该邻居
3. 重复

访问的节点序列就是**随机游走**。这产生了对图邻域结构的随机探索。

### 随机游走嵌入：核心思想

```
z_v^T · z_u  ≈  P(v 和 u 在图上的随机游走中共同出现)
```

**为何使用随机游走？**

1. **表达力** — 对相似度的灵活随机定义，同时捕获*局部*和*高阶*邻域信息。如果从 `u` 出发的随机游走以高概率访问 `v`，则认为 `u` 和 `v` 相似。

2. **效率** — 无需考虑所有 `O(|V|^2)` 节点对；只考虑实际共现于游走中的对。

### 特征学习作为优化

给定 `G = (V, E)`，通过求解以下问题学习 `f : u → R^d`（即 `f(u) = z_u`）：

```
max_f  Σ_{u ∈ V}  log P(N_R(u) | z_u)
```

其中 `N_R(u)` 是在策略 `R` 下从 `u` 出发的随机游走中访问到的节点的多重集。节点可以多次出现（它是一个多重集）。

**直觉**：给定节点 `u`，学习能够*预测*可能出现在 `u` 的随机游走邻域中的节点的嵌入。

### 随机游走优化

**逐步过程：**

1. 使用策略 `R` 从图中每个节点 `u` 开始运行短的固定长度随机游走
2. 对每个节点 `u`，收集 `N_R(u)` — 访问节点的多重集
3. 使用以下损失优化嵌入：

```
L = Σ_{u ∈ V}  Σ_{v ∈ N_R(u)}  -log P(v | z_u)
```

使用 **Softmax** 参数化 `P(v | z_u)`：

```
P(v | z_u) = exp(z_u^T · z_v) / Σ_{n ∈ V} exp(z_u^T · z_n)
```

综合起来：

```
L = Σ_{u ∈ V}  Σ_{v ∈ N_R(u)}  -log [ exp(z_u^T · z_v) / Σ_{n ∈ V} exp(z_u^T · z_n) ]
```

**问题**：对所有节点的嵌套求和导致 `O(|V|^2)` 复杂度——对大型图来说代价过高。

---

### 负采样（Negative Sampling）

**解决方案**：使用**负采样（Negative Sampling）** 近似 Softmax 归一化（噪声对比估计，NCE，Noise Contrastive Estimation 的一种形式）：

```
log [ exp(z_u^T · z_v) / Σ_{n ∈ V} exp(z_u^T · z_n) ]

≈  log σ(z_u^T · z_v)  -  Σ_{i=1}^{K} log σ(z_u^T · z_{n_i})
```

其中：
- `σ` 是 Sigmoid 函数
- `n_i ~ P_V` — 按节点度比例采样的 `K` 个负样本
- 用 `K` 个随机"负"比较替换了昂贵的完整归一化

**选择 K：**
- K 越高 → 估计越鲁棒
- K 越高 → 对负事件的偏置越高
- **实践中：K = 5 到 20**

**注意**：从技术上讲，负样本应该是不在游走中的节点；但实践中为了效率，使用任意节点。

**参考**：噪声对比估计论文 — https://arxiv.org/pdf/1402.3722.pdf

---

### 随机梯度下降（SGD）

形成目标 `L` 后，使用**随机梯度下降**最小化它：

**全梯度下降：**
1. 随机初始化所有节点 `u` 的 `z_u`
2. 迭代直到收敛：
   - 为所有 `u` 计算梯度 `∂L/∂z_u`
   - 更新：`z_u ← z_u - η · ∂L/∂z_u`（`η` 为学习率）

**随机梯度下降（SGD）** — 更实用：
1. 随机初始化所有节点的 `z_u`
2. 对每个训练样本（节点 `u`）：
   - 计算单节点损失：`L^u = Σ_{v ∈ N_R(u)} -log P(v | z_u)`
   - 计算所有维度 `j` 的梯度 `∂L^u/∂z_j`
   - 更新：`z_j ← z_j - η · ∂L^u/∂z_j`

SGD 在单个训练样本上而非整个数据集上计算梯度，使其具有更强的可扩展性。

### 随机游走总结

```
算法：随机游走节点嵌入

1. 从每个节点 u 运行短的固定长度随机游走
2. 对每个节点 u，收集 N_R(u) = 访问节点的多重集
3. 使用 SGD 优化：
   L = Σ_{u ∈ V} Σ_{v ∈ N_R(u)} -log P(v | z_u)
   （使用负采样进行近似以提高效率）
```

## 高级 GNN 主题

### DeepWalk

**DeepWalk**（Perozzi 等，KDD 2014）使用最简单的随机游走策略：
- 从每个节点运行**固定长度、无偏**的随机游走
- 每一步均匀随机选择一个邻居

**局限**：得到的相似度概念过于受限——只能捕获对邻域的简单、均匀的局部视图。

---

### node2vec：有偏随机游走（Biased Random Walk）

**node2vec**（Grover 和 Leskovec，KDD 2016）通过*灵活的、有偏的（Biased）* 二阶随机游走（Second-order Random Walk）策略对 DeepWalk 进行了推广，可以在两个极端之间进行插值（Interpolation）：

| 策略 | 类型 | 视角 |
|----------|------|------|
| **BFS**（广度优先搜索，Breadth-First Search） | 局部探索 | 邻域的微观视角（结构等价，Structural Equivalence） |
| **DFS**（深度优先搜索，Depth-First Search） | 全局探索 | 邻域的宏观视角（同质性，Homophily） |

**两个超参数：**

- `p` — **返回参数**：控制返回到前一个节点的概率
- `q` — **进出参数**：控制 BFS 类 vs. DFS 类行为的比率
  - 低 `q` → DFS 类（探索更远）
  - 低 `p` → BFS 类（保持接近源节点）

#### 有偏二阶随机游走机制（Biased Second-order Random Walk Mechanism）

当游走者刚刚经过边 `(S1, w)` 并且当前位于节点 `w` 时，到 `w` 邻居的转移概率为：

| 下一个节点 `t` | 距 `S1` 的距离 | 未归一化概率 |
|---------------|--------------------|--------------------------|
| `S1`（返回前一个） | 0 | `1/p` |
| 距 `S1` 距离为1的节点（共同邻居） | 1 | `1` |
| 距 `S1` 距离为2的节点（更远的） | 2 | `1/q` |

这是**二阶**游走，因为转移取决于*前一个*节点（而不仅仅是当前节点）。游走者"记住"了它来自哪里。

**游走类型：**
- **BFS 类**：低 `p` → 高概率返回 → 保持局部
- **DFS 类**：低 `q` → 高概率走远 → 全局探索

#### node2vec 算法

```
1. 计算随机游走转移概率（使用 p 和 q）
2. 从每个节点 u 模拟 r 次长度为 l 的随机游走
3. 使用 SGD（配合负采样）优化 node2vec 目标
```

**复杂度**：与节点数成线性关系。三个步骤都可以单独并行化。

---

### 其他随机游走变体（Random Walk Variants）

| 方法 | 核心思想 |
|--------|----------|
| **DeepWalk**（Perozzi 等，2014） | 固定长度无偏游走 |
| **node2vec**（Grover & Leskovec，2016） | 有偏二阶游走（p、q 参数） |
| **metapath2vec**（Dong 等，2017） | 基于节点属性/类型的游走（异构图） |
| **Watch Your Step**（Abu-El-Haija 等，2017） | 基于学习权重的游走 |
| **LINE**（Tang 等，2015） | 直接优化1跳和2跳游走概率 |
| **struct2vec**（Ribeiro 等，2017） | 在修改网络上运行游走；捕获结构身份 |
| **HARP**（Chen 等，2016） | 游走前的层次图预处理 |

---

### 选择方法

没有单一方法在所有情况下都占优（Goyal 和 Ferrara，2017年综述）：
- **node2vec** 在**节点分类**上表现更好
- 其他方法在**链接预测**上可能表现更好
- **随机游走方法**通常**计算效率更高**
- 关键原则：选择与下游应用相匹配的节点相似度定义

## 图 Transformer / 现代方法

### 嵌入整个图

除节点嵌入外，我们通常需要将**整个图或子图**嵌入到单个向量 `z_G` 中。

**使用场景：**
- 将分子分类为有毒 vs. 无毒
- 识别异常图
- 药物发现/分子属性预测

介绍三种方法：

---

#### 方法1：节点嵌入的求和/平均（Sum/Average of Node Embeddings）

简单但有效：
1. 运行任意图嵌入技术为 `G` 中每个 `v` 获得节点嵌入 `z_v`
2. 聚合：

```
z_G = Σ_{v ∈ G} z_v
```

或使用平均。被 **Duvenaud 等（2016）** 用于分子图分类。

---

#### 方法2：虚拟节点（Virtual Node / Super-node）

1. 引入一个**虚拟"超级节点"**，连接到（子）图中的所有节点
2. 运行标准嵌入技术——超级节点的嵌入成为 `z_G`

由 **Li 等（2016）** 提出，作为子图嵌入的通用技术。

---

#### 方法3：匿名游走嵌入（Anonymous Walk Embeddings）

参考：*Anonymous Walk Embeddings*，ICML 2018（https://arxiv.org/pdf/1805.11921.pdf）

**核心思想**：匿名游走（Anonymous Walk）移除节点身份（Node Identity）——只有重访的*模式（Pattern）*重要。

**匿名游走定义**：在匿名游走中，每个位置由该节点*第一次被访问的索引*标记。

**示例：**
- 游走：A → B → C → B → C
- 匿名游走：1 → 2 → 3 → 2 → 3

经过不同节点的两个不同具体游走如果具有相同的模式，则产生**相同的匿名游走**。

**匿名游走数量指数增长：**
- 长度3：5种匿名游走——`{111, 112, 121, 122, 123}`
- 长度7：877种匿名游走

**思路1：简单的基于频率的表示**

- 模拟 `m` 次长度为 `l` 的匿名游走
- 将图表示为游走类型上的概率分布：
  ```
  z_G[i] = 匿名游走 w_i 出现的频率
  ```
- 这给出一个固定维度的直方图向量

**需要多少次游走？**
```
m ≥ (η / ε²) · (log(2/δ) - log(δ))
```
其中：
- `η` = 长度为 `l` 的不同匿名游走数
- `ε` = 最大可接受误差
- `δ` = 失败概率

示例：对于 `l = 7`，`ε = 0.1`，`δ = 0.01` → 需要 `m = 122,500` 次随机游走。

---

**思路2：学习游走嵌入**

与其用频率表示游走，不如**学习**每种匿名游走类型 `w_i` 的嵌入 `z_i`，以及整体图嵌入 `z_G`。

**目标**：嵌入游走，使得下一次游走可以从上下文中预测（类似于 word2vec）：

```
max_{z_G, z_i}  (1/T) Σ_{t=Δ+1}^{T}  log P(w_t | w_{t-Δ}, ..., w_{t-1}, z_G)
```

其中 `T` 是采样游走的总数，`Δ` 是上下文窗口大小。

**概率模型：**

```
P(w_t | w_{t-Δ}, ..., w_{t-1}, z_G) = exp(y(w_t)) / Σ_{i=1}^{η} exp(y(w_i))

y(w_t) = b + U · cat( (1/Δ) Σ_{i=1}^{Δ} z_i , z_G )
```

其中：
- `cat(·, ·)` = 拼接
- `(1/Δ) Σ z_i` = 窗口中游走嵌入的平均
- `b ∈ R`，`U ∈ R^d` 是可学习参数（线性层）
- `η` = 所有可能匿名游走类型的数量（需要负采样）

**优化后**：`z_G` 是一个单独优化的可学习参数，编码整个图。

**`z_G` 的下游使用：**
- **选项1**：内积核 — `z_G1^T · z_G2`（图级相似度）
- **选项2**：将 `z_G` 输入神经网络进行图分类

---

### 层次嵌入（预览）

更高级的方法（将在第8讲中涵盖）：**层次聚类**图中的节点，然后对每个聚类中的节点嵌入求和/取平均。这产生了一种层次池化，捕获多尺度图结构。

## 应用

### 如何使用节点嵌入

给定节点嵌入 `{z_i}`，下游应用包括：

#### 聚类/社区检测
- 直接在嵌入空间中对向量 `{z_i}` 进行聚类（例如，k-means）

#### 节点分类
- 基于 `z_i` 预测节点 `i` 的标签

#### 链接预测
- 基于 `(z_i, z_j)` 的某个函数预测边 `(i, j)`：

| 方法 | 公式 |
|--------|---------|
| 拼接 | `f(z_i, z_j) = g([z_i, z_j])` |
| Hadamard（逐元素乘积） | `f(z_i, z_j) = g(z_i * z_j)` |
| 求和/平均 | `f(z_i, z_j) = g(z_i + z_j)` |
| L2 距离 | `f(z_i, z_j) = g(‖z_i − z_j‖₂)` |

#### 图分类
- 使用图嵌入 `z_G`（通过节点聚合或匿名游走）预测图级标签

## 关键要点

1. **图表征学习**通过从原始图结构中学习与任务无关的节点和图嵌入，消除了手动特征工程的瓶颈。

2. **编码器-解码器框架**是统一的抽象：
   - 编码器（浅层查找）：`ENC(v) = Z · v`
   - 解码器（点积）：`DEC(z_u, z_v) = z_u^T · z_v`
   - 目标：最大化在随机游走中共现的节点的点积相似度

3. **随机游走**提供了节点相似度的灵活高效定义，捕获多跳邻域信息，而无需所有 `O(|V|^2)` 对的比较。

4. **DeepWalk**：无偏固定长度随机游走——简单基线。

5. **node2vec**：带参数 `p`（返回）和 `q`（远离）的有偏二阶随机游走。在 BFS（局部、结构性）和 DFS（全局、社区）视角之间插值。

6. **负采样**（NCE 近似）将 `O(|V|^2)` 的 Softmax 归一化成本降低到每次更新 `O(K)`，使训练可行。

7. **图级嵌入**可以通过以下方式构建：节点聚合（求和/平均）、虚拟超级节点或匿名游走嵌入。

8. **匿名游走嵌入**与身份无关，捕获游走的结构模式——对于节点身份无关紧要的图分类很有用。

9. 没有单一方法普遍最优：节点相似度定义的选择应由下游任务和应用驱动。

10. 未来方向（第8讲）：使用图神经网络（深度编码器）的**层次/池化**图嵌入，克服了浅层嵌入查找的局限。
