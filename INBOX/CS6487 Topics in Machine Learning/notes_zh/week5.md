## 图机器学习简介

### 为何使用图？

- **图**是描述和分析具有关系/交互的实体的通用语言。
- 现代机器学习方法（CNN、RNN、Transformer）是为简单序列和网格设计的。**并非所有内容都能表示为序列或网格。**
- 复杂的现实世界领域具有丰富的关系结构，自然地表示为图。
- 图是机器学习中**最热门的子领域之一**，在 ICLR 等顶级会议上的存在感迅速增长。

### 许多类型的数据都是图

| 领域 | 图类型 |
|--------|-----------|
| 社交网络 | 友谊、职业、性关系网络 |
| 信息网络 | 引用网络、万维网、互联网 |
| 生物网络 | 蛋白质-蛋白质相互作用、疾病通路、调控网络 |
| 技术网络 | 计算机网络、地下交通 |
| 其他 | 知识图谱、场景图、代码图、分子、3D形状、粒子网络、食物网 |

**核心问题：** 如何利用关系结构实现更好的预测？

### 图机器学习为何困难？

- **任意大小和复杂的拓扑结构**——不像网格那样具有空间局部性
- **没有固定的节点排序或参考点**——节点未被索引
- **通常是动态的**，并且具有**多模态特征**

### 图上的表征学习

**目标：** 将节点映射到 $d$ 维嵌入，使得网络中相似的节点被嵌入到相近的位置。

传统机器学习生命周期：
```
原始数据 --> 特征工程（手动） --> 机器学习模型 --> 预测
```

表征学习自动化了特征提取：
```
原始图 --> 表征学习 --> 嵌入 --> 下游任务
```

本课程涵盖的方法：
- **传统方法：** 图元、图核
- **节点嵌入：** DeepWalk、Node2Vec
- **图神经网络：** GCN、GraphSAGE、GAT
- **应用：** 生物医学、科学、工业

### 图机器学习的应用

#### 任务类型

| 任务类型 | 描述 | 示例 |
|-----------|-------------|---------|
| 节点级 | 预测节点的属性 | 蛋白质功能、用户分类 |
| 边级 | 预测节点间缺失/新链接 | 推荐系统、药物相互作用 |
| 子图级 | 对社区或区域进行分类/预测 | 交通预测、路线预计到达时间 |
| 图级 | 对整个图进行分类或生成 | 药物发现、分子属性预测 |

#### 应用示例

**1. 蛋白质折叠（节点级）**
- **AlphaFold**（DeepMind）在计算上解决了蛋白质折叠问题。
- **图表示：** 节点 = 氨基酸，边 = 残基间的邻近关系（"空间图"）。
- 影响：仅从氨基酸序列准确预测蛋白质3D结构；对药物发现和生物科学具有变革性意义。

**2. 推荐系统（边级）**
- **PinSage**（Pinterest）：用于网络规模推荐系统的图卷积神经网络（Ying 等，KDD 2018）。
- 节点 = 用户和物品，边 = 用户-物品交互。
- 目标：学习节点嵌入 $Z_i$，使得 $d(Z_{\text{蛋糕1}}, Z_{\text{蛋糕2}}) < d(Z_{\text{蛋糕1}}, Z_{\text{毛衣}})$。
- 预测图中两个节点是否相关。

**3. 药物副作用（边级）**
- **Decagon**（Zitnik 等，Bioinformatics 2018）：用图卷积网络建模多重用药副作用。
- 节点 = 药物和蛋白质，边 = 相互作用。
- 任务：给定一对药物，预测不良副作用（多重用药）。
- 重要性：70–79岁人群中46%服用超过5种药物；许多人服用20种以上。

**4. 交通预测（子图级）**
- **预计到达时间（ETA）预测**——部署于 Google 地图。
- 节点 = 道路段，边 = 道路段之间的连通性。
- 在多个主要城市显著减少了负面 ETA 结果。

**5. 药物发现（图级）**
- **Stokes 等，Cell 2020：** 抗生素发现的深度学习方法。
- 抗生素是小分子图：节点 = 原子，边 = 化学键。
- GNN 图分类模型从候选库中预测有前景的分子。
- **分子生成/优化：** 生成具有高类药性的新型分子，或优化现有分子的理想属性。

**6. 物理模拟（图级）**
- **Sanchez-Gonzalez 等，ICML 2020：** 使用图网络学习模拟复杂物理。
- 节点 = 粒子，边 = 粒子间的相互作用。
- 图演化任务：预测图随时间的演化。

### 图的表示

#### 网络的组成部分

- **节点（顶点）** $N$：对象/实体
- **边（链接）** $E$：交互/关系
- **图** $G(N, E)$：系统

#### 选择合适的表示

表示的选择决定了可以研究的问题类型：
- 连接一起工作的个人 → **职业网络**
- 连接有性关系的个人 → **性关系网络**
- 连接互相引用的论文 → **引用网络**

构建图时的关键问题：
- **节点**是什么？
- **边**是什么？

#### 图的类型

**有向 vs. 无向：**
| | 无向 | 有向 |
|--|------------|----------|
| 链接 | 对称、互惠 | 有向弧 |
| 示例 | 合作、Facebook 好友 | 电话通话、Twitter 关注 |

**节点度（Node Degree）：**
- **无向：** $k_i$ = 与节点 $i$ 相邻的边数；平均度 $\bar{k} = \frac{2E}{N}$
- **有向：** 入度 $k_i^{in}$，出度 $k_i^{out}$；源节点：$k^{in}=0$；汇节点：$k^{out}=0$
- $\sum_i k_i^{in} = \sum_i k_i^{out} = E$（总边数）

**二部图：**
- 节点分为两个不相交的集合 $U$ 和 $V$；每条边连接 $U$ 中的节点和 $V$ 中的节点。
- $U$ 和 $V$ 是独立集合（$U$ 或 $V$ 内部没有边）。
- 示例：作者-论文、演员-电影、用户-电影、食谱-食材。
- **折叠/投影：** 投影到一个集合上（例如作者合作网络、电影共同评分网络）。

#### 图的表示形式

**邻接矩阵（Adjacency Matrix）：**
$$A_{ij} = 1 \text{ 如果从节点 } i \text{ 到节点 } j \text{ 有链接，否则 } A_{ij} = 0$$

- 无向：$A_{ij} = A_{ji}$（对称），$A_{ii} = 0$
- 有向：$A_{ij} \neq A_{ji}$（一般情况），$A_{ii} = 0$
- 度：$k_i = \sum_j A_{ij}$（无向的行求和），$k_i^{out} = \sum_j A_{ij}$，$k_i^{in} = \sum_i A_{ij}$
- 大多数真实网络是**稀疏的**：$E \ll E_{max}$；邻接矩阵充满零（例如万维网密度 $= 1.51 \times 10^{-5}$，MSN 即时通讯密度 $= 2.27 \times 10^{-8}$）。

**边列表：** 将图表示为（源，目标）对的列表。简单但难以查询邻居。

**邻接列表：** 对每个节点存储其邻居列表。对大型稀疏图更为适合；允许快速检索邻居。

**节点和边的属性：**
- 权重（例如通信频率）
- 排名（最好的朋友、第二好的朋友，……）
- 类型（朋友、亲属、同事）
- 符号（信任 vs. 不信任）
- 结构属性（共同朋友数）

**更多图类型：**
| 类型 | 描述 | 示例 |
|------|-------------|---------|
| 无权 | $A_{ij} \in \{0,1\}$ | 友谊、超链接 |
| 加权 | $A_{ij} \in \mathbb{R}$ | 合作、互联网、道路 |
| 自环 | $A_{ii} \neq 0$ | 蛋白质 |
| 多图 | 同一对节点间有多条边 | 通信、合作 |

#### 连通性

**无向图：**
- **连通：** 任意两个顶点可通过路径相连。
- **不连通：** 由两个或多个连通分量组成。
- **巨型分量：** 最大的连通分量。
- **孤立节点：** 没有边的节点。
- 不连通图的邻接矩阵呈**块对角形式**。

**有向图：**
- **强连通：** 每个节点都能到达其他所有节点，反之亦然。
- **弱连通：** 忽略边方向后是连通的。
- **强连通分量（SCC）：** 互相可达的最大子集。
  - **入分量：** 能到达 SCC 的节点。
  - **出分量：** 从 SCC 可达的节点。

## 传统图机器学习

### 概述

传统图机器学习流程：
1. 为节点/链接/图**设计特征**（手工设计，$d$ 维向量 $\in \mathbb{R}^D$）
2. 为所有训练数据**获取特征**
3. **训练机器学习模型**（随机森林、SVM、神经网络）
4. **应用模型：** 给定新的节点/链接/图，获取特征并进行预测

手工设计特征的质量是实现良好模型性能的关键。

---

### 节点级特征

**目标：** 表征节点在网络中的结构和位置。

#### 1. 节点度（Node Degree）

$$k_v = \text{与节点 } v \text{ 相邻的边数}$$

- 邻居节点的简单计数。
- 平等对待所有邻居节点（不捕获重要性）。

#### 2. 节点中心性（Node Centrality）

**特征向量中心性（Eigenvector Centrality）：**
- 如果一个节点 $v$ 被重要的邻居包围，则它是重要的。
- 递归定义：
$$c_v = \frac{1}{\lambda} \sum_{u \in N(v)} c_u$$
- 矩阵形式：$\lambda \mathbf{c} = A\mathbf{c}$ — 中心性 $\mathbf{c}$ 是 $A$ 的**特征向量**。
- 由**Perron-Frobenius 定理**，最大特征值 $\lambda_{max}$ 总是正的且唯一。
- 对应 $\lambda_{max}$ 的特征向量 $\mathbf{c}$ 被用作中心性。

**介数中心性（Betweenness Centrality）：**
- 如果一个节点位于其他许多节点对之间的最短路径上，则它是重要的。
$$c_v = \sum_{s \neq v \neq t} \frac{\#(s \text{ 和 } t \text{ 之间经过 } v \text{ 的最短路径})}{\#(s \text{ 和 } t \text{ 之间的最短路径})}$$
- 示例：在链 A-C-D-E 中，B 从 C 分支，节点 C 具有高介数。

**接近中心性（Closeness Centrality）：**
- 如果一个节点到所有其他节点的最短路径长度小，则它是重要的。
$$c_v = \frac{1}{\sum_{u \neq v} u \text{ 和 } v \text{ 之间的最短路径长度}}$$

#### 3. 聚类系数（Clustering Coefficient）

衡量 $v$ 的邻居节点的连接程度：
$$e_v = \frac{\#(\text{邻居节点间的边数})}{\binom{k_v}{2}} \in [0, 1]$$

- 分母 = $k_v$ 个邻居间可能的边数。
- 计算 $v$ 的自我网络中的三角形数。

#### 4. 图元（Graphlets）

**核心思想：** 计算节点参与的预定子图结构（图元）的数量。

**定义：**
- **导出子图：** 由顶点子集及原始图中连接这些顶点的所有边构成。
- **图同构：** 两个节点数相同且以相同方式连接的图。
- **图元：** 有根的、连通的、导出的、非同构的子图。

在最多5个节点上共有 **73 种不同的图元**（位置 0–72）。

**图元度向量（GDV，Graphlet Degree Vector）：**
- 以给定节点为根的图元的计数向量。
- 对于2–5节点的图元，GDV 有73个坐标。
- 示例：图元 $a, b, c, d$ 的 GDV $= [2, 1, 0, 2]$。

**类比：**
- 节点度 → 计算节点接触到的边数
- 聚类系数 → 计算节点接触到的三角形数
- GDV → 对许多图元类型计算节点接触到的图元数

#### 节点级特征总结

| 类别 | 特征 | 使用场景 |
|----------|----------|----------|
| **基于重要性的** | 节点度、特征向量中心性、介数中心性、接近中心性 | 预测有影响力的节点（例如社交网络中的名人用户） |
| **基于结构的** | 节点度、聚类系数、图元度向量 | 预测图中的角色（例如蛋白质-蛋白质相互作用网络中的蛋白质功能） |

### 链接级特征

**目标：** 为节点对设计特征以预测新链接。

**链接预测的两种形式：**
1. **随机缺失链接：** 随机删除一组链接并尝试预测它们。
2. **随时间产生的链接：** 给定 $G[t_0, t_0']$，预测在 $G[t_1, t_1']$ 中出现的链接。

#### 1. 基于距离的特征

- $v_1$ 和 $v_2$ 之间的**最短路径距离**。
- 局限：不能捕获邻域重叠的程度。

#### 2. 局部邻域重叠（Local Neighborhood Overlap）

捕获 $v_1$ 和 $v_2$ 之间共享的邻居节点数：

| 度量 | 公式 |
|--------|---------|
| **共同邻居** | $\|N(v_1) \cap N(v_2)\|$ |
| **Jaccard 系数** | $\frac{\|N(v_1) \cap N(v_2)\|}{\|N(v_1) \cup N(v_2)\|}$ |
| **Adamic-Adar 指数** | $\sum_{u \in N(v_1) \cap N(v_2)} \frac{1}{\log(k_u)}$ |

**局限：** 如果两个节点没有共同邻居，即使它们未来可能相连，度量也始终为零。

#### 3. 全局邻域重叠（Global Neighborhood Overlap）

使用**整个图结构**对两个节点进行评分。

**Katz 指数（Katz Index）：** 计算给定节点对之间所有长度的游走数（Walks of All Lengths）。

**关键洞察——邻接矩阵的幂：**
- $A_{uv}^{(1)}$ = 长度为1的游走数（直接邻域）= $A_{uv}$
- $A_{uv}^{(2)}$ = 长度为2的游走数 = $(A^2)_{uv}$
- $A_{uv}^{(l)}$ = 长度为 $l$ 的游走数 = $(A^l)_{uv}$

推导：
$$P_{uv}^{(2)} = \sum_i A_{ui} \cdot P_{iv}^{(1)} = \sum_i A_{ui} \cdot A_{iv} = (A^2)_{uv}$$

**Katz 指数公式：**
$$S_{uv} = \sum_{l=1}^{\infty} \beta^l A_{uv}^l, \quad 0 < \beta < 1 \text{（折扣因子）}$$

**闭合形式计算**（矩阵的几何级数）：
$$S = \sum_{i=1}^{\infty} \beta^i A^i = (I - \beta A)^{-1} - I$$

#### 链接级特征总结

| 方法 | 描述 | 局限 |
|--------|-------------|-----------|
| 基于距离 | 最短路径长度 | 不能捕获邻域重叠 |
| 局部邻域重叠 | 共同邻居、Jaccard、Adamic-Adar | 没有共同邻居时为零 |
| 全局（Katz 指数） | 计算两节点间所有游走数 | 计算成本较高 |

### 图级特征与图核（Graph-level Features and Graph Kernels）

**目标：** 表征整个图结构的特征。

#### 核方法（Kernel Methods）

- **核** $K(G, G') \in \mathbb{R}$：衡量两个图之间的相似度。
- **核矩阵** $\mathbf{K} = (K(G, G'))_{G,G'}$ 必须是半正定的。
- 存在特征表示 $\phi(\cdot)$ 使得 $K(G, G') = \phi(G)^T \phi(G')$。
- 一旦定义了核，就可以使用现成的模型（例如**核 SVM**）。

**核心思想——图的词袋（Bag-of-*）：**

朴素方法：使用节点计数的词袋——失败，因为两个不同的图可以有相同的节点计数向量。

更好的方案：使用**节点度**作为"词"。更进一步：使用**图元**或**颜色细化**。

#### 图元核（Graphlet Kernel）

**思想：** 计算图中不同图元的数量。

注意：图级特征的图元与节点级图元不同：
- 图元中的节点**不需要相连**（允许孤立节点）
- 图元是**无根的**

对于 $k=3$，有4种图元（$g_1, g_2, g_3, g_4$）。
对于 $k=4$，也有4种图元。

**图元计数向量：**
$$f_G = ((f_G)_1, (f_G)_2, \ldots, (f_G)_{n_k}) \text{ 其中 } (f_G)_i = \#(g_i \subseteq G)$$

**图元核：**
$$K(G, G') = h_G^T h_{G'}, \quad h_G = \frac{f_G}{\text{Sum}(f_G)} \text{（归一化）}$$

归一化处理不同大小的图。

**局限：** 通过枚举计算大小为 $k$ 的图元需要 $O(n^k)$ 时间；子图同构是 **NP 难**问题。如果节点度以 $d$ 为界，存在 $O(nd^{k-1})$ 算法。

#### Weisfeiler-Lehman（WL）核

**目标：** 高效的图特征描述子 $\phi(G)$。

**思想：** 使用邻域结构迭代地丰富节点词汇——节点度词袋的推广。

**算法：颜色细化（Color Refinement）**

1. 为每个节点 $v$ 分配初始颜色 $c^{(0)}(v)$。
2. 迭代地细化节点颜色：
$$c^{(k+1)}(v) = \text{HASH}\left(\{c^{(k)}(v),\ \{c^{(k)}(u)\}_{u \in N(v)}\}\right)$$
   其中 HASH 将不同输入映射到不同颜色。
3. 经过 $K$ 步后，$c^{(K)}(v)$ 总结了 $v$ 的 $K$ 跳邻域的结构。

**颜色细化示例（2步）：**
- 第0步：所有节点获得颜色1。
- 第1步：聚合邻居颜色 → 哈希 → 新颜色（例如 $\{1,11\} \to 2$，$\{1,111\} \to 3$，等）
- 第2步：再次聚合和哈希 → 更精细的颜色（例如示例中的颜色6–13）。

**WL 图特征：**
- 颜色细化后，计算具有每种颜色的节点数。
- 将图表示为**颜色词袋**向量 $\phi(G)$。

**WL 核：**
$$K(G, G') = \phi(G)^T \phi(G')$$

**计算复杂度：**
- 每步颜色细化：**与边数成线性关系**（聚合邻居颜色）。
- 只需跟踪两个图中都出现的颜色 → 至多总节点数个颜色。
- 颜色计数：与节点数成线性关系。
- **总计：与边数成线性关系**——比图元核高效得多。

**与 GNN 的联系：** WL 核与图神经网络密切相关（消息传递聚合邻居信息，类似于颜色细化）。

#### 图级特征总结

| 方法 | 表示 | 复杂度 | 关键特性 |
|--------|---------------|------------|--------------|

## 图神经网络

### 动机

传统图机器学习使用**手工设计特征**：
- 设计耗时
- 不自适应（特征是固定的，不被学习）
- 可能遗漏重要的结构模式

**图神经网络（GNN）** 从图结构和节点/边属性中自动学习特征。

### 表征学习范式

$$f: N \to \mathbb{R}^d$$

学习将每个节点映射到 $d$ 维嵌入的函数，使得网络中相似的节点被嵌入到相近的位置。

### 本课程涵盖的方法

- **GCN（图卷积网络）：** 在图上应用类似卷积的操作。
- **GraphSAGE：** 从局部邻域中采样和聚合特征。
- **GAT（图注意力网络）：** 使用注意力机制对邻居贡献加权。

> **注意：** 详细的 GNN 部分（GCN、GraphSAGE、GAT、消息传递框架）将在后续讲座中涵盖。WL 颜色细化与消息传递 GNN 之间的联系是一个关键洞察：两者都迭代地聚合邻域信息。

## 关键要点

1. **图无处不在。** 社交、生物、信息和物理系统都具有关系结构，自然地表示为图。

2. **图机器学习具有挑战性**，原因在于任意大小、没有固定节点排序以及复杂拓扑——这与图像或序列不同。

3. **图的表示很重要。** 节点、边和方向性的选择决定了可以研究的问题。

4. **真实世界的图是稀疏的。** 邻接矩阵大多是零；实践中使用边列表或邻接列表。

5. **三个级别的图机器学习任务：** 节点级、边级（链接预测）和图级预测。

6. **节点特征：**
   - *基于重要性：* 度、特征向量中心性、介数、接近。
   - *基于结构：* 度、聚类系数、图元度向量（GDV）。

7. **链接特征：**
   - *局部：* 共同邻居、Jaccard 系数、Adamic-Adar。
   - *全局：* Katz 指数（通过邻接矩阵的幂计算所有长度的游走数）。

8. **图核**支持图级分类：
   - **图元核：** 表达力强但 NP 难以计算。
   - **WL 核：** 高效颜色细化；与边数成线性关系；与 GNN 密切相关。

9. **邻接矩阵的幂：** $(A^k)_{uv}$ 计算节点 $u$ 和 $v$ 之间长度为 $k$ 的游走数——这对链接预测（Katz 指数）和图神经网络都是基础。

10. **WL 核预示着 GNN：** 两者都迭代聚合邻域信息以构建更丰富的表示。GNN 学习这种聚合；WL 核使用固定哈希。
