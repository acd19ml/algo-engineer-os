### 本轮聚焦

表征学习（Representation Learning）为什么重要，以及 XOR 为什么逼出了多层网络。

### 本轮讲解摘要

#### 1. 问题背景

早期方法依赖人工特征，模型本身只负责在这些特征上做分类。神经网络的重要变化是：不再把“特征工程”和“分类器”拆开，而是让模型自己学表示。

#### 2. 直觉层

“表示”可以理解成：把原始输入换一种坐标系或描述方式，让原来难分的问题变得容易分。

XOR 是最小反例：

- 输入是二维点 `(x1, x2)`
- 标签规则是“两个输入不同则为 1，相同则为 0”
- 在原始平面里，它不是线性可分的，所以单层线性模型做不到

多一层隐藏层之后，网络可以先把输入变换到一个新的表示空间，再在这个新空间里线性分开。

#### 3. 形式层

单层线性模型本质上是：

`y = sigma(w1 x1 + w2 x2 + b)`

它的决策边界是一条直线，因此不能表达 XOR。

两层网络则是：

`h = phi(W1 x + b1)`

`y = sigma(W2 h + b2)`

关键不是“层数变多了”本身，而是中间的隐藏表示 `h` 引入了非线性变换，使得后续线性层可以在新表示上完成分类。

#### 4. 设计动机

这就是“表征学习”的最核心动机：

- 不是直接在原始输入空间硬分
- 而是先学一个更适合当前任务的表示，再做下游预测

所以后面 CNN 学边缘/纹理/部件，Transformer 学 token 之间关系，本质都在做同一件事：学习一个更有利于任务的内部表示。

#### 5. 地图定位

这部分是第一周很多后续概念的底座：

- 表征学习解释了为什么“深度”有价值
- 但深度一上来，就会碰到梯度消失/爆炸
- 所以下一步自然会进入：为什么深层网络难训练，以及残差连接为什么关键

#### 6. 反例与边界

- 如果没有非线性激活，多层线性层叠加后仍然只是一个线性变换，依旧解决不了 XOR
- 所以“深”本身不是重点，“非线性表示能力”才是重点

### 本轮讲解摘要

#### 1. 诊断结论

用户对“XOR 需要非线性”这个结论已经有了正确直觉；当前需要补的是公式阅读能力，而不是继续堆更多新概念。

#### 2. 公式翻译

- `x`：输入。对 XOR 来说可以写成一个二维向量 `x = [x1, x2]`
- `w`：权重。可以理解成模型给每个输入维度分配的“重视程度”
- `w^T x`：本质就是点积，在二维情况下等于 `w1*x1 + w2*x2`
- `b`：偏置。可以理解成整体平移量
- `sigma(...)`：输出层上的一个函数。很多地方它特指 sigmoid，但在这里先把它理解成“最后把线性分数变成输出的函数”
- `phi(...)`：隐藏层激活函数的通用记号，比如 ReLU、tanh。它的关键作用不是名字，而是“非线性”

#### 3. 单层模型在说什么

`y = sigma(w^T x + b)`

翻译成人话就是：

“先把两个输入按权重加起来，再加一个偏置，得到一个分数；然后通过 `sigma` 变成输出。”

在二维里，`w^T x + b = 0` 对应一条直线，所以这种模型只能学到“直线切平面”的分类方式。

#### 4. 两层模型在说什么

`h = phi(W1 x + b1)`

`y = sigma(W2 h + b2)`

翻译成人话：

- 第一层先把原始输入做一次变换，得到中间表示 `h`
- 这个 `h` 不是原始坐标，而是模型自己学出来的新表示
- 第二层不再直接看原始 `x`，而是在 `h` 这个新空间里做分类

#### 5. 为什么去掉激活函数就不行

如果把 `phi` 去掉，第一层就只剩下线性变换：

`h = W1 x + b1`

代入第二层：

`y = W2 h + b2`

`= W2(W1 x + b1) + b2`

`= (W2 W1) x + (W2 b1 + b2)`

这仍然只是“一个新的权重矩阵乘以输入，再加一个新的偏置”。

也就是说：

- 两层线性层叠加
- 可以被合并成一层线性层
- 所以表达能力没有本质增加
- XOR 依然解不了

#### 6. 对“激活函数把直线变曲线”的修正

这个直觉方向是对的，但更精确的说法是：

- 激活函数不是简单地“把一条直线掰弯”
- 它让整个网络不再能被压缩成一个单独的线性变换
- 因此模型可以形成更复杂的决策边界

对 ReLU 这类激活来说，边界很多时候甚至不是光滑曲线，而是由多段线性片段拼出来的复杂形状。

### 本轮聚焦

建立两层网络的脑内图像：隐藏层不是一个整体黑盒，而是一组小判别器；每个隐藏神经元先做一次简单判断，输出层再把这些判断组合起来。

### 本轮讲解摘要

#### 1. 对第 2 题的精化

更精确的说法不是“把一条直线代入另一条直线”，而是：

- 第一层如果没有激活函数，只是在做一次仿射变换（线性变换 + 偏置）
- 第二层再接一个仿射变换
- 两个仿射变换复合后，结果仍然是一个仿射变换

所以它本质上还是“只能画一个线性决策边界的模型”。

#### 2. 两层网络的结构图像

把隐藏层想成几个并排的小神经元：

- 每个隐藏神经元都看输入 `x`
- 每个神经元先算一次 `w^T x + b`
- 再过一个激活函数，形成一个新的中间特征
- 输出层不直接看原始输入，而是看这些中间特征

因此，两层网络不是“神秘地多算一遍”，而是：

- 第一层造特征
- 第二层组合特征

#### 3. 一个适合 XOR 的最小直觉例子

对输入只考虑 `0/1` 的四个点：

- `(0,0)`
- `(0,1)`
- `(1,0)`
- `(1,1)`

可以让两个隐藏神经元分别学两个判断：

- `h1`：是否“至少有一个 1”
- `h2`：是否“两个都是 1”

那么：

- `(0,0)` -> `h1=0, h2=0`
- `(0,1)` -> `h1=1, h2=0`
- `(1,0)` -> `h1=1, h2=0`
- `(1,1)` -> `h1=1, h2=1`

这时输出层只需要学会：

- “`h1=1` 且 `h2=0` 时输出 1”

也就是：

- 有一个 1，但不是两个都是 1

这正好就是 XOR。

#### 4. 激活函数在这里扮演什么角色

激活函数让隐藏神经元的输出不再只是线性数值的平滑搬运，而是真正形成了新的、可组合的非线性特征。

没有激活函数：

- 第一层造不出这种“是否满足某条件”的特征
- 第二层拿到的仍然只是线性变换后的坐标
- 所以整体依然无法实现 XOR 这样的逻辑组合

### 本轮聚焦

修正“第一层直接让原问题线性可分”的误区，明确“线性可分”总是相对于某个表示空间而言。

### 本轮讲解摘要

#### 1. 用户这句话哪里对

对的部分是：

- 第一层确实在“造特征”
- 第二层确实在利用这些特征做分类

这已经抓到了多层网络的主干。

#### 2. 用户这句话哪里需要修正

不够精确的地方是：

- 不是说第一层自己已经把整个任务“做完了”
- 而是第一层把原始输入变成了一个更有用的表示
- 第二层再在这个新表示上做线性判别

所以更准确的话是：

“第一层先把原始输入映射成一组中间特征；如果这些特征足够有用，那么第二层就可以用线性方式把类别分开。”

#### 3. 为什么这里容易混

因为我们口头常说“变得线性可分”，听起来像是第一层独立完成了分类准备。

其实真正发生的是：

- 原始空间里，样本点位置是 `(x1, x2)`
- 经过第一层后，样本被映射到另一个空间，比如 `(h1, h2)`
- 在这个新空间里，第二层更容易用一条直线或一个线性规则把它们分开

所以“线性可分”这个说法，必须带上前提：

- 是在 `h` 空间里线性可分
- 不是在原始 `x` 空间里突然 magically 变成一条线就能分

#### 4. 用 XOR 再压一遍

第一层不是直接输出“这是 XOR 的 1 类还是 0 类”，而是先输出类似这样的特征：

- `h1`：至少有一个 1
- `h2`：两个都是 1

然后第二层根据这两个特征做组合：

- `h1=1` 说明“可能是 XOR 的正类”
- 但 `(1,1)` 也满足 `h1=1`，所以会误伤
- 再用 `h2` 把 `(1,1)` 排除掉

所以：

- 第一层提供“描述问题所需的部件”
- 第二层把这些部件拼成最终判断

#### 5. 一句定稿

对你刚才那句话的最稳版本可以写成：

“隐藏层不是直接分类，而是学习新的特征表示；如果这些表示把问题组织得更好，输出层就能用线性方式完成最终分类。”

### 本轮聚焦

解释为什么公式里写一个 `h`，但实际可以有 `h1, h2, ..., hk` 多个隐藏特征。

### 本轮讲解摘要

#### 1. 一个隐藏层不等于一个隐藏神经元

- “一层”说的是结构层级
- “一个神经元”说的是这一层里面的单元数量

所以：

- 可以有一层隐藏层，里面只有 1 个神经元
- 也可以有一层隐藏层，里面有 2 个、10 个、100 个神经元

#### 2. 为什么公式里只写一个 `h`

因为这里的 `h` 通常不是标量，而是整个隐藏层输出组成的向量。

例如如果隐藏层里有 2 个神经元，那么：

`h = [h1, h2]`

更一般地，如果有 `k` 个隐藏神经元，那么：

`h = [h1, h2, ..., hk]`

所以公式

`h = phi(W1 x + b1)`

里的 `h`，常常默认就是“整层隐藏表示”。

#### 3. XOR 为什么自然会用两个隐藏单元举例

因为我们刚才构造了两个中间特征：

- `h1`：至少有一个 1
- `h2`：两个都是 1

这意味着隐藏层至少要有两个输出位置来承载它们，于是就可以把隐藏层写成：

`h = [h1, h2]`

然后输出层再根据这个二维表示做分类。

#### 4. 维度上怎么理解

对 XOR：

- 输入 `x = [x1, x2]`，所以输入维度是 2
- 如果隐藏层有 2 个神经元，那么隐藏表示 `h` 的维度也是 2
- 输出层再把这个二维隐藏表示映射成最终输出 `y`

也就是说，流程是：

`x(2维) -> h(2维) -> y(1维)`

#### 5. 如果隐藏层真的只有一个神经元呢

那就只有一个隐藏特征，例如：

`h = [h1]`

这时模型表达能力会弱很多。对我们这里的 XOR 直觉构造来说，1 个隐藏单元不够，因为它没法同时表达：

- “至少有一个 1”
- “两个都是 1”

这两个彼此不同的中间条件。

### 本轮聚焦

把 XOR 两层网络的公式写成“按维度展开”的版本，让用户知道每个矩阵/向量到底对应哪一层的哪些参数。

### 本轮讲解摘要

#### 1. XOR 这个网络的维度

如果我们用：

- 2 个输入：`x = [x1, x2]`
- 1 个隐藏层
- 隐藏层里 2 个神经元：`h = [h1, h2]`
- 1 个输出：`y`

那么整个网络就是：

`x(2维) -> h(2维) -> y(1维)`

#### 2. 第一层参数 `W1` 和 `b1`

第一层要把 2 维输入变成 2 维隐藏表示，所以：

- `W1` 可以看成一个 `2 x 2` 的参数表
- `b1` 是一个 `2 维` 偏置向量

把它按“每个隐藏神经元各自有一组权重”展开，就是：

`h1 = phi(w11*x1 + w12*x2 + b1_1)`

`h2 = phi(w21*x1 + w22*x2 + b1_2)`

这里你可以把它读成：

- 第一个隐藏神经元拿一套自己的权重，看 `(x1, x2)`
- 第二个隐藏神经元再拿另一套自己的权重，看同样的 `(x1, x2)`

所以 `W1` 不是神秘矩阵，它只是把这两组权重打包在一起。

#### 3. 第二层参数 `W2` 和 `b2`

第二层要把 2 维隐藏表示变成 1 个输出，所以：

- `W2` 是一个 `1 x 2` 的权重
- `b2` 是一个标量偏置

展开后就是：

`y = sigma(v1*h1 + v2*h2 + b2)`

意思很直接：

- 输出层看 `h1`
- 输出层也看 `h2`
- 然后把它们组合成最终输出

#### 4. 整个网络按“人话”重读一遍

第一层：

- 神经元 1 从 `(x1, x2)` 里提取一个特征 `h1`
- 神经元 2 从 `(x1, x2)` 里提取另一个特征 `h2`

第二层：

- 把 `h1` 和 `h2` 这两个特征组合起来，得到最终预测 `y`

#### 5. 为什么这一步重要

后面你再看到：

`h = phi(W1 x + b1)`

就不需要把它当成一坨抽象符号了，而可以自动翻译成：

- 这一层里有若干个神经元
- 每个神经元都有一组自己的权重和偏置
- 它们一起组成下一层的表示向量

- 方向：参数该增大还是减小
- 幅度：该改很多还是只改一点

#### 3. 为什么深层网络麻烦

- 输出层的误差
- 要一层一层往回传
- 才能传到很前面的层

如果这些因子大多：

- 小于 1，连乘很多次后就会越来越小
- 大于 1，连乘很多次后就会越来越大

这就是梯度消失和梯度爆炸的直觉来源。

#### 4. 梯度消失的直觉图像

于是前面的层会出现：

- 梯度非常小
- 参数几乎不更新
- 学得极慢，甚至像没学一样

这就是梯度消失。

#### 5. 梯度爆炸的直觉图像

反过来，如果每过一层都被放大一点：

- 小误差一路被越放越大
- 传到前面时变成极大的更新信号

于是会出现：

- 参数更新过猛
- 训练震荡
- loss 不稳定，甚至直接发散

这就是梯度爆炸。

### 本轮讲解摘要

#### 1. 先纠正一个定义

梯度不是神经元参数本身。

更准确地说：

- 参数是 `W`、`b`
- 梯度是“损失对这些参数的导数”或“变化率”

也就是：

- 参数：你要调的旋钮
- 梯度：告诉你这个旋钮往哪边拧、拧多大

#### 2. 那些“小于 1 的东西”是什么

直觉上，它们是**每一层对误差信号的局部缩放因子**。

来源通常有两类：

- 激活函数在当前点的导数
- 线性层/权重对前一层信号的局部影响

你现在先不用背完整公式，只记住：

- 误差要从输出层传回前面层
- 每过一层，都要乘一个“这一层会把误差信号放大还是缩小”的局部因子

这个因子有时小于 1，有时大于 1。

#### 3. 这些因子怎么得到

它们来自**链式法则**。

如果一个量是层层套起来的：

- 最后输出依赖中间变量
- 中间变量又依赖更前面的变量

那么“最后输出对最前面变量的影响”就要拆成一串局部影响相乘。

最小例子：

如果：

- `a = 2x`
- `y = 3a`

那么 `y` 对 `x` 的影响，不是凭空来的，而是：

- `a` 对 `x` 的影响是 `2`
- `y` 对 `a` 的影响是 `3`

所以 `y` 对 `x` 的影响就是 `3 * 2 = 6`

神经网络只是把这个“层层相乘”的过程扩展到了很多层。

#### 4. 为什么越小就越学不到东西

因为前面层真正拿来更新参数的，是它们收到的梯度。

如果回传到前面层时，梯度已经非常小，比如接近 0，那么：

- 这个参数改一点，loss 看起来几乎不变
- 优化器就只会给它一个极小的更新
- 训练很多轮，它也动得很慢

所以不是“参数本身变小了就学不到”，而是：

- **参数收到的更新信号变小了**
- 于是参数更新几乎停滞

#### 5. 回传里是乘法，更新里是加减法

这两个过程你已经开始分出来了，这是对的。

要明确区分：

- **反向传播求梯度**：核心是链式法则，所以会出现很多项相乘
- **参数更新**：拿到梯度之后，再做加减法更新

最常见的更新形式是：

`参数新值 = 参数旧值 - 学习率 * 梯度`

所以：

- 回传时：是乘法链条在决定梯度有多大
- 更新时：是减法在真正修改参数

#### 6. 一句最稳的总结

梯度消失不是说“参数越来越小”，而是说：

“误差信号在多层回传中被很多局部因子不断缩小，导致前面层拿到的梯度接近 0，于是这些层几乎学不到东西。”

- 预测得准，loss 小
- 预测得差，loss 大

它是训练时用来衡量“模型现在有多错”的一个数。

所以训练的目标，本质上就是：

- 调参数
- 让 loss 变小

#### 3. loss 长什么样

最简单地想，loss 就是一个标量，也就是**一个数**。

例如：

- 预测很好：loss 可能是 `0.1`
- 预测很差：loss 可能是 `3.2`

它不是向量，不是矩阵，在训练里通常就是一个单独的数，用来概括当前这批样本上的错误程度。

#### 4. 什么情况下 loss 会降

当你更新参数的方向是对的，并且步子大小合适时，loss 就会下降。

也就是：

- 参数往“让预测更接近真实答案”的方向调
- 而不是往相反方向调

如果方向错了，loss 会升；
如果步子太大，也可能震荡甚至升高。

这句话最朴素的意思是：

- 如果我把某个参数稍微改一点点
- loss 会增加还是减少
- 增减得有多明显

例如对某个参数 `w`：

- 如果把 `w` 稍微增大一点，loss 明显升高
- 那说明当前 `w` 往增大的方向不好

反过来：

- 如果把 `w` 稍微减小一点，loss 下降
- 那说明往减小的方向更好

所以梯度本质上是在回答：

- 这个参数现在该往哪边走？
- 走多大合适？

#### 6. 为什么训练闭环是 `参数 -> 预测 -> loss -> 梯度 -> 参数更新`

1. 先拿当前参数 `W, b` 去做预测
2. 把预测和真实答案比较，算出一个 loss
3. 根据 loss 反向传播，算每个参数的梯度
4. 用梯度更新参数
5. 再用新参数继续下一轮

所以 `loss` 是中间的桥：

- 没有 loss，就不知道模型错得多严重
- 没有梯度，就不知道该怎么改参数

`loss` 是“模型当前错得有多厉害”的分数；
`梯度` 是“这个分数对每个参数有多敏感”的信号；
训练就是不断利用梯度调整参数，让 loss 下降。

### 本轮讲解摘要

#### 1. 最小链式法则例子

先看两步函数：

- `a = 2x`
- `y = 3a`

那么：

- `a` 对 `x` 的变化率是 `2`
- `y` 对 `a` 的变化率是 `3`

于是 `y` 对 `x` 的变化率就是：

`dy/dx = (dy/da) * (da/dx) = 3 * 2 = 6`

这就是链式法则最核心的意思：

- 如果影响要经过多步传递
- 总变化率就是各步局部变化率相乘

#### 2. 套到多层网络

如果把一个很深的网络简化成：

- 第 1 层输出依赖第 0 层
- 第 2 层输出依赖第 1 层
- ...
- 最后 `loss` 依赖最末层输出

那么最前面某个参数的梯度，就会像这样：

`dL/dw = (dL/dh_n) * (dh_n/dh_{n-1}) * ... * (dh_2/dh_1) * (dh_1/dw)`

你现在不用管每一项具体怎么写，先抓住结构：

- **很多个局部导数在连乘**

#### 3. 为什么会消失

如果这串乘积里的很多项都像：

- `0.8`
- `0.5`
- `0.2`

那么：

- `0.8 * 0.5 = 0.4`
- 再乘 `0.2 = 0.08`
- 再多来几层，就会很快接近 0

所以前面层最终拿到的梯度会非常小。

#### 4. 为什么会爆炸

反过来，如果很多项都像：

- `1.2`
- `1.5`
- `2.0`

那么连乘后会迅速变大。

于是前面层拿到的梯度可能非常大，更新会很不稳定。

#### 5. 为什么激活函数和权重都会影响这个过程

因为这些局部导数通常就来自：

- 激活函数在当前点的导数
- 线性层权重对前一层的局部影响

所以：

- 激活函数选得不好
- 权重初始化不合适
- 网络太深

都会让这串连乘更容易失控。

#### 6. 一句最稳的总结

梯度消失 / 爆炸的最小形式层原因是：

“在深层网络里，前面层的梯度要通过链式法则由很多局部导数连乘得到；这些局部因子若普遍小于 1，就会让梯度指数级缩小，若普遍大于 1，就会让梯度指数级放大。”

#### 2. 初始化到底在解决什么

初始化不是为了让模型一开始就会做题，而是为了让模型在训练刚开始时：

- 前向传播的信号不要一层层放大或缩小得太离谱
- 反向传播的梯度也不要一层层炸掉或消失

所以初始化的目标不是“随机一下就行”，而是：

- 让每层输入输出的尺度尽量稳定

#### 3. 为什么不能把权重设得太大

如果初始权重很大：

- 前向传播时，激活值容易变得很大
- 反向传播时，局部导数链条也更容易被放大

后果可能是：

- 梯度爆炸
- 激活函数进入饱和区
- 训练一开始就很不稳定

#### 4. 为什么也不能把权重设得太小

如果初始权重很小：

- 每层输出都被压得很弱
- 信号一层层传下去会越来越小

后果可能是：

- 前向传播的表示变得很弱
- 反向传播的梯度更容易消失
- 前面层几乎学不到东西

#### 5. 最直观的工程目标

所以好的初始化，本质上是在追求：

- 每层信号“别越传越大”
- 也“别越传越小”

也就是尽量让：

- 前向的激活分布比较稳定
- 反向的梯度分布也比较稳定

#### 6. 为什么会出现 Xavier 和 Kaiming

因为研究者发现：

- 不同激活函数的性质不同
- 比如 ReLU 会把一部分输出直接变成 0

所以初始化不能一刀切，必须考虑：

- 这一层有多少输入连接
- 激活函数会怎样改变信号尺度

这就是后面 Xavier、Kaiming 初始化出现的原因。

#### 7. 一句最稳的总结

初始化是在训练开始前，先把权重放到一个“既不太大也不太小”的合理范围里，让深层网络的前向和反向信号都不至于一开始就失控。

### 本轮讲解摘要

#### 1. 初始化公式背后在守什么

前一轮已经知道，初始化的核心目标是：

- 不让前向信号越传越大或越传越小
- 不让反向梯度越传越大或越传越小

更形式一点说，就是在尽量守住：

- 每层输出的**方差/尺度**

让它跨层传播时不要失控。

#### 2. Xavier 初始化在说什么

Xavier 初始化的核心条件可以写成：

`n * Var(w) ≈ 1`

这里你先只这样理解：

- `n`：这一层能看到多少个输入连接
- `Var(w)`：权重分布有多“散”

这个式子想表达的是：

- 如果输入连接很多，那每个权重就该更小一些
- 否则很多输入叠加起来，输出尺度会太大

所以 Xavier 的核心思想是：

- 根据输入连接数，自动缩放初始权重
- 让线性层的输出方差尽量稳定

它更适合：

- 线性激活
- 或者比较对称的激活函数，比如 tanh / sigmoid 这类场景

#### 3. 为什么 ReLU 不能直接照搬 Xavier

因为 ReLU 有一个很关键的性质：

- 负数直接变成 0
- 平均会砍掉一大块信号

直觉上可以理解成：

- 有效通过去的信号变少了
- 如果还按 Xavier 那套尺度来初始化，信号会更容易一路变小

#### 4. Kaiming 初始化在补什么

Kaiming 初始化的核心条件可以写成：

`n * Var(w) ≈ 2`

和 Xavier 相比，它把尺度再抬高一点。

直觉上是在补偿：

- ReLU 会“截掉一半左右信号”

所以 Kaiming 初始化的核心思想是：

- 给 ReLU 网络更合适的初始权重尺度
- 尽量抵消 ReLU 带来的信号缩小问题

#### 5. 你现在不必死记的地方

这轮你不用背完整推导，也不用强记：

- 为什么是 1
- 为什么是 2

你现在只要抓住：

- Xavier：给线性/对称激活用，目标是稳住信号尺度
- Kaiming：给 ReLU 用，因为 ReLU 会砍掉一部分信号，所以要做额外补偿

#### 6. 一句最稳的总结

Xavier 和 Kaiming 都是在做“按层宽度来缩放初始权重”，区别只在于：

- Xavier 假设信号大体完整传递
- Kaiming 额外考虑了 ReLU 会把一部分信号置零，因此把初始尺度调得更适合 ReLU

- 梯度又可以通过
- 相关参数仍然会被更新

所以更准确的说法是：

- 不是“这个神经元永远不学”

- 如果某个神经元长期、几乎对所有样本都落在负区间

- 这个神经元几乎不再参与学习

- 训练刚开始那一刻的参数尺度

但训练一旦开始：

- 参数会不断更新
- 每一层输出的分布也会不断漂移

所以即使一开始初始化很好，训练过程中信号尺度仍然可能慢慢失控。

#### 2. 归一化在解决什么

归一化的核心目标是：

- 把中间激活值拉回一个更稳定、可控的范围

直觉上可以理解成：

- 某一层输出今天突然特别大
- 或特别小
- 那就先做一次规范化，再送进后面的层

这样做的好处是：

- 后面的层不需要总是适应剧烈漂移的输入分布
- 训练会更稳定

#### 3. 为什么这会帮助深层网络

因为前面我们已经知道，深层网络最怕的是：

- 信号尺度跨层积累失真
- 梯度也跟着变得不稳定

归一化就在中间不断“扶正”这些尺度。

所以它可以帮助：

- 让激活别太大
- 也别太小
- 让梯度传播更平稳
- 让训练能用更大的学习率

#### 4. 一个直觉类比

你可以把深层网络想成很多工位串联的流水线。

如果前一站送来的零件尺寸忽大忽小：

- 后面每一站都很难工作

归一化像是在中间加一个校准站：

- 不管前面漂成什么样
- 先把尺度校回到一个更稳定的范围
- 再交给下一层

#### 5. BN 和 LN 现在先怎么记

这轮你先不需要分太细。

先只记：

- BN（Batch Normalization）：在 batch 这个维度上做规范化，CNN 里常见
- LN（Layer Normalization）：在特征维度上做规范化，Transformer 里常见

它们形式不同，但共同目的都是：

- 稳定训练过程中的激活分布

#### 6. 一句最稳的总结

初始化是在训练开始前把权重放到合理范围；归一化是在训练过程中不断把中间激活拉回更稳定的尺度，从而让深层网络更容易训练。

### 本轮聚焦

解释归一化里的“分布异常”不是异常检测，而是激活值的中心和尺度漂移；同时解释均值/方差和不同统计维度的意义。

### 本轮讲解摘要

#### 1. 归一化里说的“异常”是什么

这里不要把“异常”理解成数据清洗里的离群点。

归一化更关心的是：

- 激活值整体是不是偏得太远
- 尺度是不是太大或太小
- 训练过程中这些统计量是不是一直漂

比如一层输出从：

`[-1, 0, 1]`

变成：

`[100, 120, 140]`

后面的层看到的输入尺度就完全变了。

#### 2. 均值的意义

均值表示这一组数的中心位置。

如果均值很大或很小，说明这一组激活整体偏向某一边。

例如：

- `[-1, 0, 1]` 的中心在 0 附近
- `[100, 120, 140]` 的中心在 120 附近

后者对下一层来说就是完全不同的输入范围。

#### 3. 方差的意义

方差表示这一组数分散得有多开。

方差太小：

- 大家都挤在一起
- 特征区分度弱
- 梯度也可能弱

方差太大：

- 数值跨度很大
- 后续层更容易出现不稳定

归一化通常做的是：

- 减去均值，让中心回到 0 附近
- 除以标准差，让尺度回到 1 附近

#### 4. 从不同维度收集统计量的不同意义

对 `[B, F]`：

BN 是：

- 固定一个特征
- 看这个 batch 里所有样本在这个特征上的分布

它的意义是：

- 让每个特征在一批样本里的尺度稳定

LN 是：

- 固定一个样本
- 看这个样本内部所有特征的分布

它的意义是：

- 让每个样本自己的表示向量尺度稳定

#### 5. 加不加归一化看起来有什么区别

不加归一化时，训练中可能出现：

- 每层激活均值和方差不断漂
- loss 下降慢
- 对学习率更敏感
- 深层网络更容易梯度不稳定

加归一化后，通常会看到：

- 中间激活尺度更稳定
- loss 更平滑地下来
- 可以用更大的学习率
- 深层网络更容易训练

#### 6. 一句最稳的总结

均值告诉你这一组激活的中心漂到哪里；方差告诉你这一组激活散得多开。BN 和 LN 的区别是选择不同的一组数来算这些统计量：BN 稳定“某个特征在一批样本中的分布”，LN 稳定“某个样本自己的整条特征向量”。

### 本轮讲解摘要

#### 1. 问题背景

前面已经讲过：

- 深层网络表达能力强
- 但梯度要跨很多层传播
- 即使有初始化和归一化，特别深的网络仍然很难训练

残差连接就是进一步给信号和梯度开一条更直接的通路。

#### 2. 普通层在学什么

普通网络里，一个块通常要直接学习：

`H(x)`

也就是：

- 输入 `x`
- 经过一堆变换
- 输出目标映射 `H(x)`

如果这个块其实只需要“稍微改一下输入”，那直接学整个 `H(x)` 可能反而困难。

#### 3. 残差块在学什么

残差连接把输出写成：

`y = F(x) + x`

这里：

- `x` 是原始输入，走捷径直接保留下来
- `F(x)` 是网络块学到的修正量
- 输出是“原输入 + 修正”

所以残差块不是强迫网络直接学完整的 `H(x)`，而是让它学：

`F(x) = H(x) - x`

也就是：

- 目标输出和原输入之间的差值

#### 4. 为什么这更容易

如果某一层其实不需要大改输入，只需要保留原信息，那么：

- 普通网络要学会完整复制 `x`
- 残差网络只要让 `F(x)` 接近 0

于是：

`y = 0 + x = x`

这就让“什么也不改，直接传下去”变得很容易。

#### 5. 为什么对梯度有帮助

因为 `y = F(x) + x` 里有一条直接的 `+ x` 路径。

反向传播时，梯度不必完全穿过复杂的 `F(x)` 变换，也可以沿着这条恒等路径往回传。

直觉上：

- 普通深层网络像一条很长很弯的路
- 残差连接给它加了一条直达通道

#### 6. 一句最稳总结

残差连接的核心不是“多加一个加号”，而是让网络默认可以保留输入，只学习必要的修正；同时给梯度提供更直接的回传路径，从而让非常深的网络更容易训练。

#### 2. 第一题的补全

说“不是梯度选择走哪条路”，是因为计算图里两条依赖同时存在：

- `y` 通过 `F(x)` 依赖 `x`
- `y` 也通过 `+x` 直接依赖 `x`

反向传播时不是二选一，而是：

`dy/dx = dF(x)/dx + 1`

两部分梯度贡献相加。

#### 3. 残差连接小节最终表述

最准确的当前版本是：

`F(x)` 继续学习需要的修正量；`+x` 直接保留输入并提供恒等梯度项。反向传播时，来自 `F(x)` 的梯度和来自 `x` 的梯度相加，因此深层网络更不容易把梯度完全传没。

### 本轮聚焦

理解深度可分离卷积为什么能大幅降低计算量。

### 本轮讲解摘要

#### 1. 问题背景

前面几节主要解决的是：

- 深层网络怎么训练

现在进入高效网络，问题变成：

- 模型能不能在手机、边缘设备上便宜地跑

标准卷积很强，但计算量大。

#### 2. 标准卷积在同时做两件事

标准卷积同时做：

- 空间模式提取：看局部区域，比如 `3x3`
- 通道混合：把多个输入通道混成多个输出通道

所以它一次性处理：

- 高度 `H`
- 宽度 `W`
- 输入通道 `Cin`
- 输出通道 `Cout`
- 卷积核大小 `K x K`

计算量大致是：

`H * W * Cin * Cout * K * K`

#### 3. 深度可分离卷积的核心想法

它把标准卷积拆成两步：

1. Depthwise Conv
2. Pointwise Conv

也就是：

- 先每个通道自己做空间卷积
- 再用 `1x1` 卷积做通道混合

#### 4. Depthwise Conv 做什么

Depthwise Conv 是：

- 每个输入通道单独用一个 `K x K` 卷积核
- 不跨通道混合

所以它主要负责：

- 每个通道内部的空间模式提取

计算量约为：

`H * W * Cin * K * K`

#### 5. Pointwise Conv 做什么

Pointwise Conv 是：

- 用 `1x1` 卷积
- 在每个空间位置上混合通道

它主要负责：

- 跨通道信息融合
- 从 `Cin` 变成 `Cout`

计算量约为：

`H * W * Cin * Cout`

#### 6. 为什么更便宜

标准卷积一次性做空间和通道混合：

`H * W * Cin * Cout * K * K`

深度可分离卷积分两步：

`H * W * Cin * K * K + H * W * Cin * Cout`

它便宜是因为：

- 昂贵的 `K*K` 空间卷积不再对每个输出通道都重复做
- 只在每个输入通道内部做一次
- 通道混合交给便宜的 `1x1` 卷积

#### 7. 一句最稳总结

深度可分离卷积把“空间特征提取”和“通道混合”拆开：depthwise 负责每个通道内部看局部空间，pointwise 负责跨通道组合，因此比标准卷积便宜很多。

`X = [[1, 2, 3],`

`K = [[a, b],`

`[[1, 2],`

- 同一个小卷积核
- 在不同位置复用
- 每个位置做局部加权求和

标准卷积的最小理解是：用一个可学习的小窗口在输入特征图上滑动，每个位置对局部区域做加权求和，从而提取局部空间模式。

### 本轮讲解摘要

#### 1. 标准卷积的核心成本

标准卷积总计算量：

`H * W * Cout * Cin * K * K`

贵在：

- 每个输出通道都要跨所有输入通道做 `KxK` 空间卷积

也就是说，`K*K` 这部分被重复了 `Cout` 次。

#### 2. Depthwise Conv 做什么

Depthwise Conv 的做法是：

- 每个输入通道自己配一个 `KxK` 卷积核
- 只在本通道内部做空间卷积
- 不跨通道混合

所以如果有 `Cin` 个输入通道，就有 `Cin` 个 `KxK` 卷积核。

计算量：

`H * W * Cin * K * K`

这一步输出仍然大致有 `Cin` 个通道。

#### 3. Pointwise Conv 做什么

Depthwise 之后还没有跨通道融合。

所以接一个 `1x1` 卷积，也叫 Pointwise Conv。

它在每个空间位置上看所有通道，用 `1x1` 权重做通道混合。

如果要从 `Cin` 变成 `Cout`，计算量是：

`H * W * Cin * Cout`

#### 4. 两步合起来

深度可分离卷积总计算量：

`H * W * Cin * K * K + H * W * Cin * Cout`

对比标准卷积：

`H * W * Cout * Cin * K * K`

节省来自：

- 空间卷积 `K*K` 不再为每个输出通道重复做
- 通道混合用便宜的 `1x1` 来做

#### 5. 一句最稳总结

标准卷积把“空间卷积”和“通道混合”绑在一起做；深度可分离卷积先用 depthwise 单独做每个通道的空间卷积，再用 pointwise 做通道混合，因此计算量大幅下降。

- 用户新问题：
  - 用 `1x1` 卷积和原本标准卷积直接混合，效果有什么区别？

### 本轮聚焦

区分：

- 标准卷积：空间卷积和通道混合在同一步完成
- Depthwise：只做每个通道内部的空间卷积
- Pointwise：只在同一空间位置做通道混合

### 本轮讲解摘要

#### 1. Depthwise 为什么不混通道

Depthwise Conv 的定义就是：

- 每个输入通道只配一个自己的 `KxK` 卷积核
- 这个卷积核只看本通道
- 不看其他通道

所以它不可能混通道。

如果输入是 RGB：

- R 通道只用 R 自己的 `KxK`
- G 通道只用 G 自己的 `KxK`
- B 通道只用 B 自己的 `KxK`

三个通道之间没有相加混合。

#### 2. Pointwise 为什么能混通道

Pointwise Conv 是 `1x1` 卷积。

它在某一个空间位置上，看这个位置的所有通道值。

例如某个位置上有：

`[r, g, b]`

一个 `1x1` 输出通道可以学：

`v1*r + v2*g + v3*b`

这就是通道混合。

它不看周围空间邻域，只看同一个位置上的通道向量。

#### 3. 标准卷积里的直接混合是什么

标准卷积在生成一个输出位置时，会同时看：

- R 通道的 `KxK` 邻域
- G 通道的 `KxK` 邻域
- B 通道的 `KxK` 邻域

然后一次性全部加权求和。

所以标准卷积是：

- 空间邻域处理
- 跨通道混合

在同一步里一起做。

#### 4. 和 Depthwise + Pointwise 的区别

Depthwise + Pointwise 把标准卷积拆成两步：

第一步：

- 每个通道各自做空间卷积
- 不混通道

第二步：

- 每个位置用 `1x1` 混合通道
- 不看空间邻域

所以区别不是最终完全不能混通道，而是：

- 标准卷积：同一步同时混空间和通道
- 深度可分离卷积：先空间、后通道，拆开做

#### 5. 代价与表达的取舍

标准卷积更自由：

- 每个输出通道都可以为每个输入通道学习不同的 `KxK` 空间模式

Depthwise + Pointwise 更便宜：

- 空间卷积只对每个输入通道做一次
- 输出通道的差异主要通过 `1x1` 通道组合形成

因此它省计算，但也相当于对标准卷积做了结构约束。

#### 6. 一句最稳总结

标准卷积是在每个输出通道里同时学习“怎么在空间上看”和“怎么混通道”；深度可分离卷积把这两件事拆开，Depthwise 只看每个通道自己的空间邻域，Pointwise 只在同一位置混合通道。

### 给用户的检查题

1. 为什么 `1x1` 卷积能混通道，但不能看空间邻域？
2. 标准卷积和 Depthwise + Pointwise 的核心区别，是不是“一个能混通道，一个不能混通道”？如果不是，更准确该怎么说？

## Session 50 - Depthwise Separable Conv 的效果取舍

时间：2026-04-26 22:56:51 HKT

### 用户回答

1. 因为 `1x1` 不看周围邻域。它只看同一个空间位置上的通道向量。
2. 标准卷积里，每个输出通道都能为每个输入通道学习不同的 `3x3` 空间模式。Depthwise + Pointwise 里，空间卷积先按输入通道单独做一次，后面的输出通道主要通过 `1x1` 对这些通道结果重新组合出来。

### 用户追问

具体效果上，标准卷积一定比 Depthwise + Pointwise 更好吗？

### 本轮讲解要点

准备区分三件事：

- 表达能力：标准卷积更自由，理论上能表示更复杂的空间与通道联合模式。
- 计算成本：Depthwise + Pointwise 明显更省参数和计算。
- 实际效果：不只由表达能力决定，还受数据量、模型规模、过拟合、训练稳定性、硬件效率和任务类型影响。

### 讲解记录

结论：标准卷积不一定在实际效果上总是比 Depthwise + Pointwise 更好。

更准确地说：

- 单看同一层的表达能力，标准卷积更强。
- 单看计算量和参数量，Depthwise + Pointwise 更省。
- 单看最终模型效果，不一定谁稳赢，要看任务、数据、模型宽度、训练方式和硬件。

标准卷积的一层权重形状是：

```text
Cout x Cin x K x K
```

这表示：每个输出通道、对每个输入通道，都可以学习一套自己的 `KxK` 空间模式。

Depthwise + Pointwise 则拆成：

```text
Depthwise: Cin x K x K
Pointwise: Cout x Cin x 1 x 1
```

其中 Depthwise 阶段有一个重要限制：

- 对同一个输入通道，它先只学一套空间卷积结果。
- 后面的不同输出通道，只能通过 `1x1` 对这些结果重新加权组合。
- 所以它不是完全等价于标准卷积，而是对标准卷积加了结构约束。

用一个例子：

输入通道 `Cin=3`，输出通道 `Cout=8`，卷积核 `K=3`。

标准卷积参数量：

```text
8 * 3 * 3 * 3 = 216
```

Depthwise + Pointwise 参数量：

```text
Depthwise: 3 * 3 * 3 = 27
Pointwise: 8 * 3 = 24
Total: 51
```

所以它省参数、省计算，但代价是表达受限制。

### 类比

标准卷积像是：每个输出通道都有一整套独立的观察方式，可以分别决定“怎么看空间邻域”和“怎么混合颜色/通道”。

Depthwise + Pointwise 像是：先让每个输入通道各自做一次固定的空间加工，然后再用 `1x1` 把这些加工结果调配成不同输出通道。

### 对“哪个更好”的判断

标准卷积更适合：

- 预算足够。
- 数据和任务复杂。
- 需要强表达能力。
- 不太受移动端/实时速度限制。

Depthwise + Pointwise 更适合：

- 需要轻量化。
- 需要减少计算和参数。
- 移动端、边缘设备、实时推理。
- 可以接受一定表达能力约束，或者通过堆叠更多层来补偿。

### 用户回答

1. 参数量对比：

```text
标准卷积参数量:

```

### 反馈

这两个回答是正确的。

关键点：

- 参数量减少来自把 `Cout * Cin * K * K` 拆成 `Cin * K * K + Cout * Cin`。
- 表达限制来自 Depthwise 阶段：每个输入通道先只产生一份空间处理结果，后续输出通道主要通过 `1x1` 重新组合这些结果。

### 下一步

进入 MobileNet 为什么采用 Depthwise Separable Conv：它不是单纯降低能力，而是在边缘设备上用结构约束换取更低计算成本。

### 本轮讲解：为什么 MobileNet 使用这种拆分

MobileNet 的核心目标不是在无限算力下追求最高精度，而是在手机、边缘设备、实时应用里获得足够好的精度和很低的计算成本。

标准卷积的问题：

- 表达能力强。
- 但 `Cout * Cin * K * K` 很贵。
- 当特征图的 `H, W` 很大时，计算量会被进一步放大。

Depthwise Separable Conv 的策略：

- 用 Depthwise 负责每个通道内部的空间模式。
- 用 Pointwise 负责同一位置上的跨通道混合。
- 承认“不是每个输出通道都需要为每个输入通道重新学习一套完整空间卷积”。

所以它的隐含假设是：

> 空间模式提取和通道混合可以近似拆开做。

这个假设不总是完美成立，但很多视觉任务里足够有用。

如果用普通语言描述：

- 标准卷积：每个输出通道都派一整组专家去看所有输入通道的局部区域。
- Depthwise + Pointwise：先让每个输入通道各自整理自己的局部信息，再让输出通道从这些整理好的信息里重新配比。

因此它不是“偷工减料”，而是一个架构选择：

- 牺牲部分表达自由度。
- 大幅降低参数和计算。
- 在轻量模型里获得更好的速度/精度平衡。
