# Initialization and Normalization

## 它要解决什么问题

深层网络的根本训练困难是**信号尺度跨层失控**：前向激活和反向梯度都要跨很多层传播，每经过一层都会被局部因子缩放——连乘下来要么消失要么爆炸（详见 `KNOWLEDGE/ml/gradient-flow-deep-networks/`）。

初始化和归一化是同一类问题在**两个不同时刻**的修补：

- **初始化**管"训练开始前"——把参数放到合理尺度
- **归一化**管"训练过程中"——把中间激活持续拉回稳定范围

它们不互相替代——这是关键。

## 朴素直觉为什么不够：初始化不是"为了让模型一开始就会做题"

很多人把初始化理解成"给模型一个好的起点"，听起来好像越接近最优解越好。

不对。初始化的真正目标不是任务性能，而是**让每层输入输出的尺度从一开始就接近 1**——也就是**前向激活别越传越大或越传越小，反向梯度也别越传越大或越传越小**。

如果一开始权重就太大：

- 前向：激活值越来越大，激活函数容易进饱和区
- 反向：局部导数链条容易被放大 → 梯度爆炸

如果一开始权重就太小：

- 前向：信号一层层被压弱
- 反向：梯度更容易消失

初始化的工程目标，**就是让深层网络在第一步前向 / 反向时尺度不立即失控**。

## Xavier：按输入连接数缩放权重

Xavier 初始化的核心条件可以写成：

```
n · Var(w) ≈ 1
```

这里 `n` 是这一层能看到多少个输入连接，`Var(w)` 是权重分布有多"散"。

直觉：如果输入连接很多，那每个权重就该更小一些，否则很多输入叠加起来输出尺度会太大。所以 Xavier **根据输入连接数自动缩放初始权重**——目标是让线性层的输出方差大致稳定在 1。

适合：线性激活、或者 tanh / sigmoid 这类**比较对称的激活**。

## 反事实：为什么 ReLU 不能直接照搬 Xavier

ReLU 有一个 Xavier 假设里没有的性质：**负数直接变成 0**。

如果某一层在线性变换后得到的值大致围绕 0 对称分布，那么过 ReLU 之后：

- 大约一半为负 → 直接变成 0
- 大约一半为正 → 继续传

**有效信号被平均砍掉一半左右**（不是严格 50%，是直觉近似）。如果还按 Xavier 那套尺度，前向信号会一路被压弱。

## Kaiming：补偿 ReLU 砍掉的一半信号

Kaiming 把守的目标抬高到：

```
n · Var(w) ≈ 2
```

把 Xavier 的 1 改成 2——直观上就是**抵消 ReLU 砍掉一半信号的效应**：原信号过 ReLU 损失一半，那初始权重的尺度补到原来的两倍，整体尺度就稳定了。

所以两个的关系：

- **Xavier**：假设信号大体完整传递（线性 / 对称激活）
- **Kaiming**：额外考虑 ReLU 会把一部分信号置零，所以把初始尺度调高一档

## 反事实：为什么"只有初始化"不够，还需要归一化

初始化只能控制**训练第一步前**的尺度。但训练一旦开始：

- 参数会不断更新
- 每一层输出的分布会跟着漂移

即使一开始初始化得很好，**训练过程中信号尺度仍然可能慢慢失控**。

归一化的角色就是在中间不断"扶正"这些尺度——它管"过程中"，初始化管"开始时"。

## 归一化在做什么：减均值、除标准差

不管什么归一化，本质都是同一个动作：

1. 选一组数
2. 对这组数算均值和方差
3. 用 `(x - mean) / std` 把这组数拉回稳定尺度

**减均值**解决"中心漂移"——原来一组数的中心可能在 120，拉回到 0 附近。
**除标准差**解决"尺度不统一"——原来一组数典型波动幅度可能是 50，拉回到 1 附近。

注意一个容易混的点：**归一化不是要让中心在 1 附近**，是要让中心在 0 附近、标准差在 1 附近。这里的 "1" 指的是**尺度**（标准差），不是中心。

### 为什么减均值还不够、必须再除标准差

举三组例子，都减均值后中心都在 0：

- 原始 `[1, 2]` → 减均值 → `[-0.5, 0.5]`
- 原始 `[10, 20]` → 减均值 → `[-5, 5]`
- 原始 `[100, 200]` → 减均值 → `[-50, 50]`

中心都在 0，**但尺度差 100 倍**——下一层看到的输入幅度还是完全不同。再除以各自的标准差（0.5、5、50），三者都变成 `[-1, 1]`，**尺度才真正统一**。

只减均值能让后面层看到 "中心" 一致，但 "幅度" 还会让权重和梯度在不同层处于完全不同的范围——影响不亚于不归一化。

## BN vs LN：固定谁、沿谁收集

BN 和 LN 的形式差别看起来很像、容易记混。**用一个心法看**：先确定形状 `[B, F]`（B 是 batch、F 是特征），然后两个问题：

1. 我**固定**什么？
2. 我**沿哪个方向**收集一组数来算均值和方差？

| | 固定的 | 沿哪个方向收集 | 在问什么 |
|---|---|---|---|
| **BN** | 特征 (f) | batch 维度 (b) | "这一批样本在第 f 个特征上的分布是什么样？" |
| **LN** | 样本 (b) | feature 维度 (f) | "这个样本自己的整条特征向量分布是什么样？" |

具体到一个矩阵：

```
X = [[1, 10, 100],
     [2, 20, 200],
     [3, 30, 300],
     [4, 40, 400]]
shape = [4, 3]
```

- BN 处理 feature 1 → 收集 `[1, 2, 3, 4]`（不同样本，同一特征）
- LN 处理 sample 1 → 收集 `[1, 10, 100]`（同一样本，所有特征）

BN 稳定的是 "**某个特征在 batch 内的分布**"；LN 稳定的是 "**某个样本自己的特征向量尺度**"。

这个心法的好处是**形状变了也能套**——比如 `[B, T, D]`（语言模型）或 `[B, C, H, W]`（图像），先确定每个轴是什么，再套"固定谁、沿谁收集"，两种归一化的差异立刻清楚。

## Open Questions

- ReLU "砍掉一半信号" 是依赖 "输入大致围绕 0 对称分布" 的假设。**如果激活分布偏了，Kaiming 的补偿系数 2 还成立吗**？这一点直觉上说要随分布调整，但工程实践里 Kaiming 仍然广泛用且效果好——是因为分布偏移不大、还是因为后续的归一化把这个偏差吃掉了？
- BN 在训练时沿 batch 收集统计量，**推理时这个 batch 不存在**——通常用训练阶段累积的 running mean/var。这个推理时的"代理统计量"和训练时是否对齐，对模型的最终行为影响多大？这是 BN 在小 batch / 不同分布下表现不稳的根源之一，本节点没展开。
- 归一化通常配合可学习的 **gamma / beta** 参数把数值再线性变换回去——这等于是允许模型"反归一化"。如果可以反归一化，那归一化的好处到底体现在哪？这个 trade-off 我没完全想清。
