# Gradient Flow in Deep Networks

## 它要解决什么问题

深层网络的表达能力来自层数。但层数变多之后训练就不动了——这不是优化器的问题，是反传信号本身在传播过程中已经退化。要理解为什么"深"在训练上反而是个负担，得先看清楚反向传播在数学上做了什么。

## 朴素直觉为什么不够

很多人对 "梯度消失" 的第一反应是 "参数变小了"。错。

参数（W、b）和梯度（∂L/∂W）是两个东西：

- 参数是模型记住的东西
- 梯度是 loss 在告诉每个参数 "你该往哪走、走多敏感"

更新规则是 `W_new = W_old - lr · ∂L/∂W`。如果 ∂L/∂W 接近 0，W 几乎不动——但 W 本身没变小。所以 "梯度消失" 准确说是 "**前面层收到的更新信号接近 0**"，不是参数缩小。混淆这一点之后，后面所有讨论都会跑偏。

## 为什么会接近 0：链式法则是连乘不是连加

反传计算 ∂L/∂W_l（第 l 层的梯度）时，必须从 loss 一路链式法则乘回 l 层。每经过一层，乘上一个局部导数（激活函数导数 × 权重）。所以：

```
∂L/∂W_1 = (loss 对最后一层的偏导) × (各层局部导数的连乘)
```

**关键观察：这是一个连乘，不是连加。**

设有 20 层，每层局部导数 ≈ 0.5（比如 sigmoid 在饱和区，导数接近 0.25）。连乘 20 层 ≈ `0.5^20 ≈ 1e-6`。前面层收到的就是这个量级的信号——等于不更新，训练再多轮也几乎不动。

反过来，每层局部导数 ≈ 2，连乘 20 层 ≈ `2^20 ≈ 1e6`。这时候参数被推得过猛，loss 震荡甚至发散——这就是梯度爆炸。

所以 "梯度消失" 和 "梯度爆炸" 不是两个独立现象，是**同一个连乘机制的两个方向**：连乘小于 1 的东西就消失，大于 1 的就爆炸。

## 为什么前向传播没这个问题

前向也是逐层计算，也是某种 "连乘"——每层一次矩阵乘法。直觉上，前向稳就该反向稳，但事实不是。

区别在于：**前向乘的是激活值**（实际信号的数值），**反向乘的是激活函数的导数**。

ReLU 在前向不放大不缩小信号（正半轴是恒等映射）。但 ReLU 的**导数**在负半轴是 0、正半轴是 1——这个导数才是反传时被连乘的东西。如果有一半神经元落在负半轴，反传时这一层就被 0 截断了。

sigmoid 更明显：前向输出有界 [0, 1]，看起来 "很稳"，但导数 `σ'(x) = σ(x)(1-σ(x))` 在两端饱和处趋近于 0。反传一连乘就把梯度乘没了。所以 **前向稳定 ≠ 反向稳定**。

## 由此引出三条互不替代的修补方向

如果根本问题是 "局部导数连乘的量级失控"，那么所有缓解方法本质上都在改这个连乘——只是改的时刻不同：

- **初始化**（Xavier、Kaiming）：训练第一步前就让每层输入输出尺度接近 1。管的是 "起点"。
- **归一化**（BN、LN）：训练过程中持续把激活拉回稳定范围。管的是 "过程中"——因为参数更新会让激活分布漂移，初始化只够开局。
- **残差连接**：让梯度有一条**不经过非线性**的旁路（identity 路径上局部导数恒为 1）。管的是 "回传路径"——前两条改的是连乘的每一项，残差改的是 "有没有一项一定是 1"。

这三条路不互相替代。它们解决的是同一问题在不同时刻 / 不同路径上的表现。

## Open Questions

正文三条修补方向都在 "改连乘的量级"。但还有一条更直接、性质完全不同的做法：**梯度裁剪**（gradient clipping）——它不预防问题，而是问题发生后强制把梯度范数压回阈值之内。

它不在三条路里，因为：
- 修的**时刻**不同（事后救而不是预防）
- 修的**对象**不同（整体范数，不是局部导数）

它的 trade-off 我没想清：

- 当单个 batch 梯度极大时（比如某个 outlier 样本），强行缩小整体范数，是不是把 "少数极端方向" 和 "多数正常方向" 一起按比例压缩了？这是不是破坏方向信息？
- 实践中 clip-by-norm 和 clip-by-value 选择依据是什么？什么场景该用哪个？
