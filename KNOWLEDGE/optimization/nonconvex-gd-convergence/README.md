# Nonconvex GD Convergence

## 它要解决什么问题

GD 单步下降已经搞清楚（详见 `KNOWLEDGE/optimization/l-smooth-gradient-descent/`）：在 L-光滑、`alpha <= 1/L` 时，每一步都能保证 loss 至少下降一个和 `||∇f(x_k)||²` 成正比的量。

但单步下降不能直接告诉你 "**训练 T 步之后能到哪里**"。这里有个根本问题——深度学习的 loss 通常**不是凸的**。

凸 vs 非凸的差别决定了能给出什么样的保证：

- **凸函数**（像一个碗）：局部最低点 = 全局最低点；GD 可以保证收敛到全局最优
- **非凸**：可能有多个谷、局部极小、鞍点、平台。从任意点出发的 GD **不一定能到全局最优**——它可能卡在局部极小或鞍点

所以非凸情形下，**保证对象必须降级**——不再保证 "到达全局最优"，而是保证 "**到达某种'没有明显下降方向'的位置**"。

## 朴素方案为什么不够：`min` 而不是 `||∇f(x_T)||`

非凸保证的标准形式是：

```
min_{0 <= k < T} ||∇f(x_k)||² <= 2 · (f(x_0) - f*) / (alpha · T)
```

注意是 `min` 而不是 `||∇f(x_T)||`（最后一步的梯度）。

朴素的反应可能是 "**最后一步应该是最优的，为什么写 min**"？

不对。理论保证的不是 "最后一步最好"——它保证的是 "**前 T 步里至少有一步梯度小**"。这个差别要紧。

为什么是 "至少一步"？因为单步下降只能告诉你**每一步**的下降量正比于 `||∇f(x_k)||²`，把 T 步求和只能控制**总和或平均**——平均小，那至少有一个不超过平均的，但不一定是最后那个。

`||∇f(x_T)||` 不一定最小——loss 单调下降不代表 `||∇f||` 单调下降。**梯度小这个事件可能出现在中间任何一步**，理论保证只能说 "**它一定出现在前 T 步里某一步**"。

## 推导：从单步下降到多步求和

起点是单步下降不等式（`alpha <= 1/L`）：

```
f(x_{k+1}) <= f(x_k) - (alpha/2) · ||∇f(x_k)||²
```

移项：

```
(alpha/2) · ||∇f(x_k)||² <= f(x_k) - f(x_{k+1})
```

含义：**第 k 步的梯度平方，最多消耗这一小步实际下降的 loss 预算**。

对 `k = 0, 1, ..., T-1` 求和：

```
(alpha/2) · Σ ||∇f(x_k)||² <= Σ [f(x_k) - f(x_{k+1})]
                                = f(x_0) - f(x_T)    （右边 telescoping，中间项相消）
```

再用 `f*` 是下界（`f(x_T) >= f*`）：

```
f(x_0) - f(x_T) <= f(x_0) - f*
```

所以：

```
(alpha/2) · Σ ||∇f(x_k)||² <= f(x_0) - f*
```

最后用"最小值不超过平均值"：把右边 `Σ` 写成 `T · 平均`，整理出：

```
min_{0 <= k < T} ||∇f(x_k)||² <= 2 · (f(x_0) - f*) / (alpha · T)
```

## 直觉：把 f(x_0) - f* 当成"总下降预算"

理论的精神可以这样解读：

- `f(x_0) - f*` 是**总下降预算**——loss 最多还能下降这么多
- 每一步如果梯度大，就要消耗一部分预算（`(alpha/2)·||∇f(x_k)||²`）
- 总预算有限，所以**不可能每一步梯度都很大**
- T 越大，平均可下降空间越小 → "至少有一步梯度小" 的保证越强

这个 "**预算用尽必然变小**" 的逻辑是非凸保证的核心直觉。

## 反事实：T 增大让保证变强，不等于训练实际更好

容易踩的细节：T 越大，`min` 上界越小——但**这是数学保证变强，不是实际行为变好**。

具体说：

- **正确读法**："训练 T 步后，前 T 步中那个最优一步的梯度范数，受这个上界控制；T 越大上界越小"
- **错误读法**："T 越大，最后一步训练越好"

最后一步可能比某个中间步更糟——理论保证不约束它。

## L 不是超参数：L 是函数的性质

容易混的另一点：L 既然在公式里出现，是不是也是要调的超参数？

**不是**。L 是目标函数本身的性质（梯度变化速度的上界），由函数决定。

比如一维 `f(x) = 5x²`：

- 一阶导数 `f'(x) = 10x`，梯度变化速度就是 10 → `L = 10`
- 二阶导数 `f''(x) = 10` → 也对应 L = 10

**真实神经网络通常不精确算全局 L**——太贵且通常被某个尖锐区域支配，过于保守。L 在理论里是固定常数，但实践不直接用。

## 凸情形对比：函数值 O(1/k) 收敛

如果是凸的（比起仅光滑），保证可以更强：直接控制函数值距离最优值的差距。

```
f(x_k) - f* <= (L / 2k) · ||x_0 - x*||²
```

也就是 `f(x_k) - f* = O(1/k)`——**误差上界按 1/k 速率下降**。

具体直觉。设上界常数 C：

| k | error 上界 |
|---|---|
| 10 | 10 |
| 100 | 1 |
| 1000 | 0.1 |

**要把误差上界缩小 10 倍，通常要把步数增加 10 倍**——这是**次线性收敛**。

次线性 vs 线性收敛的差距：线性收敛是 `error <= C·ρ^k`（ρ < 1），比 `1/k` 快很多；GD 在凸+光滑下只能保证次线性。

## Open Questions

- 非凸保证只说"前 T 步至少一步梯度小"——**实践中我们关心的是 "训练完之后模型够好"**。这两个目标的 gap 在深度学习里到底有多大？经验上，loss 收敛 ≠ 梯度范数变小，因为可能进入很平的区域 loss 不动但 ||∇f|| 也不算特别小。
- 鞍点（saddle point）是非凸地形的另一个挑战——梯度为 0 但不是极小。**普通 GD 在鞍点的逃逸率是怎样的**？这条线连接动量、随机扰动等技术，本节点没碰到。
- 凸 + 光滑下 GD 是 `O(1/k)`，但 **Nesterov 加速可以做到 `O(1/k²)`**——加速的本质是什么？为什么深度学习里加速效果不像理论上那么明显？这条线和动量、自适应优化器联通，待单独节点处理。
