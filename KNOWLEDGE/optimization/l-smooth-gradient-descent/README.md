# L-Smooth Gradient Descent

## 它要解决什么问题

梯度下降的更新规则是：

```
x_new = x - alpha · ∇f(x)
```

直觉上，沿负梯度方向走一步，loss 应该会下降。但**真正的问题**是：**走完一步之后，loss 是否真的下降**？

如果学习率 `alpha` 太大，即使方向一开始是下降方向，也可能**跨过最低点**——loss 不降反升、震荡甚至发散。

要回答 "alpha 多大才安全"，光看梯度方向不够。还得知道**函数会不会弯得太快**——因为函数弯得急的话，沿当前梯度方向走远一点，那个位置的实际函数值可能完全不是线性外推预测的样子。

`L`-光滑性就是用来**限制函数弯曲速度**的工具。

## 把 L 和学习率分清楚：路况 vs 油门

很多人第一反应是把 `L` 当成"更好的学习率"。这是**根本性的误解**，必须纠正。

| | 是什么 | 谁决定 |
|---|---|---|
| **`alpha`（学习率 / 步长 / lr）** | 算法选择的超参数 | 人或调参程序 |
| **`L`（光滑常数）** | 目标函数本身的性质——梯度变化速度的上界 | 函数 |

类比：**`L` 是路况，`alpha` 是油门**。路越弯（L 越大），油门越不能踩大。

`L` 不会"自动改变" `alpha`——理论告诉我们看到 `L` 大时**应该**主动把 `alpha` 设小，但 `alpha` 仍然是你显式选的。

容易混的旁支：`lr / 学习率 / 步长 / alpha` 是同一个东西在不同语境的叫法——代码里叫 `lr`、中文叫学习率、几何上叫步长、数学符号 `alpha` 或 `η`。它们没有任何冲突。

## L-光滑性的定义：为什么是"任意两点"

`L`-光滑性的定义：

```
||∇f(x) - ∇f(y)|| <= L · ||x - y||
```

读法：**不管你拿函数上的哪两个点，只要它们距离近，它们的梯度差也不能大得离谱**。`L` 是这个差距比的上界。

容易踩的坑：会把 `x` 和 `y` 误以为只是 "**当前点和下一点**"。**不是**——定义要求**任意两个**参数位置都满足这个上界。

但在分析梯度下降一步的时候，**可以把 `y` 选成下一步**：

```
y = x - alpha · ∇f(x)
```

这时光滑性就给了我们一个工具，分析"从 x 走到 y 之后 loss 怎么变"。

## 反事实：用一维二次函数看清楚 alpha L 的作用

设一维函数：

```
f(x) = (L/2) · x²
```

它的梯度是：

```
∇f(x) = L · x
```

> 为什么写 `(L/2)·x²` 而不是直接 `L·x²`？后者求导是 `2L·x`，多出一个 2。写 `(L/2)x²` 是为了求导刚好是 `Lx`，让后续公式干净。

梯度下降一步：

```
x_new = x - alpha · L · x = (1 - alpha · L) · x
```

具体数字。设 `L = 10`：

| `alpha` | `1 - αL` | `x_new` | 行为 |
|---|---|---|---|
| 0.05 | 0.5 | `0.5x` | 离 0 越近，**稳定** |
| 0.2 | -1 | `-x` | 跳到另一边、距离不变、**震荡** |
| 0.3 | -2 | `-2x` | 跳到另一边、距离变大、**发散** |

稳定条件大致是：

```
alpha < 2/L
```

这就是为什么 "L 越大，安全 alpha 越小"。

## 下降引理：把"线性预测 + 弯曲修正"拆开

L-光滑函数满足下降引理：

```
f(y) <= f(x) + <∇f(x), y-x> + (L/2) · ||y-x||²
```

读法：**从 x 走到 y 之后的 loss，最多不超过这个上界**。三部分：

- `f(x)`：当前 loss
- `<∇f(x), y-x>`：用当前梯度做的**线性预测**（如果只相信当前梯度，预测移动到 y 后 loss 怎么变）
- `(L/2)·||y-x||²`：**弯曲修正项**——因为函数会弯，纯线性预测可能太乐观，加一个保守惩罚

## 代入 GD 一步：单步下降不等式

把 `y = x - alpha·∇f(x)` 代入下降引理。`y - x = -alpha·∇f(x)`，所以：

线性项：

```
<∇f(x), -alpha·∇f(x)> = -alpha · ||∇f(x)||²    （下降项，非正）
```

二次修正项：

```
(L/2) · ||-alpha·∇f(x)||² = (L/2) · alpha² · ||∇f(x)||²    （惩罚项，非负）
```

合起来：

```
f(x - alpha·∇f(x)) <= f(x) - alpha · (1 - L·alpha/2) · ||∇f(x)||²
```

**关键观察**：右边比 `f(x)` 小（也就是单步必下降），当且仅当：

```
1 - L·alpha/2 > 0    →    alpha < 2/L
```

这正好和前面用一维二次函数推出的稳定条件**对得上**。

## 为什么 alpha = 1/L 是常用稳妥选择

代入 `alpha = 1/L`：

```
1 - L·alpha/2 = 1 - 1/2 = 1/2
```

注意：**不是 0**——这是手算容易出错的地方。

所以单步下降变成：

```
f(x_new) <= f(x) - (alpha/2) · ||∇f(x)||²
```

只要梯度非零，loss 至少下降一个和 `||∇f(x)||²` 成正比的量。

`alpha = 1/L` 之所以是"理论上常用稳妥选择"，是因为这时候下降系数刚好 `1/2`，对应着 "**走一半的稳定空间**"。

## 反事实：线性项什么时候才是"下降项"

容易踩的细节：`<∇f(x), y-x>` **不是永远是下降项**。

只有当 `y - x` 选成 `-alpha·∇f(x)`（也就是负梯度方向）时，内积才一定非正：

```
<∇f(x), -alpha·∇f(x)> = -alpha · ||∇f(x)||²
```

如果 `y - x` 是别的方向，这一项可能是正数。所以**这个下降不等式依赖于 GD 这一步走的是负梯度方向**。

## 实践中是不是必须先算 L

理论分析里经常假设知道 `L`，从而选 `alpha <= 1/L`。但**真实深度学习里通常不会精确算全局 L**——太难、太贵，而且非凸网络的全局 L 可能非常保守（被某个尖锐区域支配，导致整体安全步长极小）。

实践中通常用：

- 经验学习率
- 学习率搜索
- learning rate scheduler
- Adam / RMSProp 等自适应优化器
- warmup

所以**不是 "训练前必须先算 L"**，而是 "**L 给出了为什么学习率不能乱设太大的理论解释**"。

## Open Questions

- L-光滑性给的是**全局上界**。深度网络的 loss 景观局部曲率差异极大——某些方向特别陡，某些方向特别平。**全局 L 受最陡方向支配，导致安全步长被压得很小**。这正是为什么 Adam / Adagrad 这类按方向自适应的方法在实践里更优。但 L-光滑性的全局 vs 局部 trade-off 我没完全想清。
- 强凸 vs 仅光滑的收敛速率差距很大（`O(ρ^k)` vs `O(1/k)`）。深度学习里基本都不强凸，但局部曲率经常类强凸——**这种"近似强凸性"的工程意义和理论保证之间的 gap 是什么**？这条线和条件数有关，本节点未碰到。
