# Continue Pretrain vs Finetune: When Each is Needed

## 它要解决什么问题

面试常被问："什么时候需要做继续预训练，什么时候直接微调就够了？"

很多人做大模型落地，**上来就想着微调**——拿到数据整理成问答对，跑个 LoRA 或全参数微调，完事。

**但有些场景下，跳过继续预训练直接微调，就是给自己挖坑**。

要做出正确决策，得先看清两件事的根本差别。

## 朴素直觉为什么不够：知识 vs 行为是两件事

继续预训练（continue pretraining）和微调（finetune）解决的是**根本不同的问题**：

| | 学习类型 | 在教什么 |
|---|---|---|
| **继续预训练** | 无监督 | 拿一堆语料喂给模型，让它继续做"下一个 token 预测"——**模型在阅读、在扩充自己的知识库** |
| **微调** | 有监督 | 给模型输入-输出问答对——**告诉它"遇到这种输入应该输出这个"**——模型在学怎么完成特定任务 |

**一个是让模型"知道更多"，一个是让模型"会做事"**。

**核心判断标准**：

- **模型懂不懂**（知识储备）和 **会不会做**（行为模式）是两回事
- **如果模型已经懂了，你只是让它学会怎么做——微调就够**
- **如果模型不懂，你光教它怎么做——它学会了格式和风格，但内容还是不靠谱**

## 反事实：直接微调够用的场景

任务满足两个条件，直接微调通常够：

1. **模型已具备完成任务所需的知识**
2. **你只是需要让模型换一种方式输出**——遵循特定格式或适配特定风格

### 例 1：电商客服机器人

业务是退换货——涉及的知识就是退换货政策、常见问题处理流程。**这些对通用大模型不是新知识，它见过大量类似内容**。

要做的只是让回复**风格符合公司规范、语气更专业、格式更统一**。

→ **直接用问答对微调就行**，甚至 LoRA 就够。更简单的是 **RAG** 或 prompt + few-shot 例子注入风格。

### 例 2：结构化 JSON 输出

模型本来就懂 JSON——你只是告诉它"每次回答都用这个 schema"。

→ **行为模式的调整，直接微调足够**。

## 反事实：必须先做继续预训练的场景

**当模型的知识储备不足以支撑你的任务时**——最典型的场景是**垂直领域**。

### 例：医疗问答系统

涉及大量专业术语、疾病机理、药物相互作用。**通用大模型在预训练阶段见过一些医疗内容，但深度不够**。

如果你拿医疗问答对去微调——**能解决问题吗？很难**。

因为微调教的是"怎么回答"，但**模型底层的知识不够**——他可能学会了回答的格式和风格，**但内容还是不靠谱**。

类比：**让一个没学过医的人背了一堆问答模板**——他能背下来，**但稍微换个问法就露馅了**。

正确做法：

1. **先做继续预训练**——拿大量医疗文献、教材、论文喂给模型，让它真正读懂医学知识、把知识底座打牢
2. **再用问答对微调**——让它学会怎么组织这些知识来回答问题

### 其他典型场景

- **小语种或特殊语言**：大部分开源模型预训练数据以英文和中文为主。如果要处理泰语、越南语，**模型见过的数据量可能不够**——这时直接微调效果很差，**因为模型对这门语言的基本理解都不够**
- **时效性知识**：模型有知识截止日期。如果任务涉及大量新知识（最新法规、行业动态），**光靠微调注入不进去**——需要用新文本数据做继续预训练

## 怎么判断你属于哪种情况：两个测试方法

### 方法 1：直接问模型

在不做任何微调的情况下，**用你领域的专业问题去问模型，看回答质量**：

- **回答能达到大致正确的内容，只是格式不对、风格不符合要求、不够精确** → 知识底座够，直接微调就行
- **回答驴唇不对马嘴，或一本正经地胡说八道，或对专业术语明显不理解** → 知识底座不够，**需要先做继续预训练**

### 方法 2：困惑度（Perplexity）测量

**量化方法**：拿你领域的文本跑一下模型的困惑度。

**困惑度越高，说明模型对这类文本越陌生**。

如果困惑度明显高于模型在通用文本上的表现 → **大概率需要继续预训练**。

## 反事实：继续预训练的四个经典坑

继续预训练**不是拿数据跑一遍就完事**了。

### 坑 1：灾难性遗忘

用领域数据做继续预训练，模型学会了新东西，**但可能忘掉了原来会的东西**。

例：用纯医疗数据训练 → 医疗知识强了，**但通用对话能力下降，甚至基本的语言能力都变差**。

**解决方案**：**混合训练**——领域数据里**混入一定比例的通用数据**。通常建议**领域数据占比不要超过 50%**——具体比例靠实验调。

### 坑 2：数据质量参差不齐

继续预训练用的数据量很大，**很难做到每条都人工审核**。但如果数据里有大量噪声 / 重复 / 格式错乱的内容，**模型学到的就是这些噪声**。

**解决方案**：花时间做数据清洗——去重、去除乱码、低质量文本。**宁可数据量少一点，也要保证质量**。

### 坑 3：训练不充分或过度

**继续预训练多少步是个问题**：

- 训练不够：知识没学进去
- 训练太久：容易过拟合或遗忘

**解决方案**：**边训边评估**——监控困惑度和下游任务的表现，找到平衡点。

### 坑 4：和微调阶段的衔接

继续预训练之后，**模型的知识更新了，但可能输出风格变得奇怪**——因为预训练阶段只是在学预测下一个 token，**没有学会按指令对话**。

**解决方案**：继续预训练之后**通常还需要做一轮微调，甚至重新做对齐**——让模型的输出回到正常的对话模式。

## 决策框架总结

需不需要继续预训练？**核心看一件事——模型现有的知识储备够不够支撑你的任务**？

```
知识够、行为不对   →   直接微调
知识不够          →   先继续预训练，再微调
```

**判断方法**：

- 直接问模型，看它懂不懂你的领域
- 用困惑度量化对该领域文本的"陌生程度"

**继续预训练的注意事项**：

- 灾难性遗忘 → 混合数据
- 数据质量要把控
- 训练步数要调
- 之后还要做对齐

**最常见的错误是跳过继续预训练**——因为觉得麻烦、成本高。**但如果你的领域确实需要，跳过这一步省下来的时间，会在后面调微调效果的时候加倍还回来**。

## Open Questions

- **混合训练里"领域数据占比 50%"** 是经验法则——具体最优比例和领域 / 模型 / 数据规模的依赖关系是什么？没有看到系统化的研究。
- **困惑度** 作为"陌生度"的代理指标有局限——同样困惑度的两类文本，对模型的训练贡献可能完全不同。这条线和数据质量评估方法相关，本节点没展开。
- **继续预训练之后再做 SFT 是否需要重新做对齐**？这个工程链条具体怎么走？SFT-RLHF-DPO 顺序在这种场景是否需要调整？
