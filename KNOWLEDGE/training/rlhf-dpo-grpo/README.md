# RLHF → DPO → GRPO: The Alignment Subtraction Path

## 它要解决什么问题

面试常被问 "RLHF 和 DPO 有什么区别？"

**大部分人上来就说一句 "DPO 去掉了 Reward Model"，然后就没了**——这样答**等于告诉面试官**"我只看了博客标题"。

要把这道题答透，得用一条主线把 **RLHF / DPO / GRPO** 三个东西串起来——**关键是搞清楚每一步到底在解决上一步的什么问题，以及它背后的数学直觉是什么**。

整条主线本质上是**对齐范式在做减法**——**追问"对齐这件事，哪些环节是真正必要的？哪些是可以被简化掉的"**。

## 起点：RLHF 标准流程的 4 步 + 4 模型

### 第 1 步：预训练

拿大规模语料让模型学到语言能力。**但此时它"只会接话，不会听话"**。

### 第 2 步：SFT（监督微调）

拿人工写的高质量 prompt-response 对教模型按指令回答问题。**到这一步模型能干活了，但它不知道什么叫"好"、什么叫"不好"**。

### 第 3 步：训练 Reward Model

同一个 prompt 让 SFT 模型生成多个回答，**让人类标注员排序哪个好哪个差**——用这些偏好数据基于 **Bradley-Terry 模型**训一个打分器。

Bradley-Terry 模型的意思很简单：**人类选 A 不选 B 的概率，正比于 A 和 B 奖励分数之差的 sigmoid**。这样你就可以用标注数据做极大似然把 Reward Model 学出来。

### 第 4 步：PPO 优化

用 Reward Model 的打分作为奖励信号，通过 PPO 算法**调整策略模型的输出分布让它尽量往高分方向靠**——同时**加一个 KL 散度约束**不让它偏离参考模型太远。

具体来说，PPO 会：

- 维护一个 **value model（也叫 critic）** 估计当前状态的价值
- 用**实际奖励和价值估计的差值**计算 advantage
- 用 **clip 机制**限制每一步的策略更新幅度，防止训崩

这一套跑下来效果确实好——**ChatGPT 用的 GPT-3.5 就是这么训出来的**。

## 反事实：RLHF 的代价是什么

这套范式**问题也很明显**——你**同时需要维护四个模型**：

1. 策略模型
2. 参考模型
3. 奖励模型
4. 价值模型

**显存压力巨大**。**PPO 本身的超参数也非常难调**。**Reward Hacking 问题也很常见**——**模型学会了"骗 Reward Model 拿高分"，而不是真的在变好**。

**核心矛盾**：**RLHF 的训练范式太重了**——工程复杂度和训练稳定性都是瓶颈。

这就是 DPO 出场的动机。

## DPO：用一个数学变换把 Reward Model 等价消掉

### 出发点

**能不能不训 Reward Model，直接用偏好数据优化策略**？

要理解 DPO 为什么能做到，**得先看一个关键的数学事实**。

### 推导链：从 KL 约束的闭式解到 BT 模型的常数项消除

在 RLHF 的目标函数里：

- **要最大化期望奖励**
- **同时最小化跟参考策略的 KL 散度**

这个**带 KL 约束的优化问题**其实**有闭式解**——**最优策略可以写成：参考策略乘以奖励函数的指数，再除以一个配分函数做归一化**。

把这个形式**反过来变换一下**——**奖励函数可以用最优策略和参考策略的对数概率差来表示，再加上一个只跟 prompt 有关的常数项**。

**关键来了**：把这个表达式**带回 Bradley-Terry 偏好模型**——你会发现**那个常数项在"好回答和坏回答的差值"里直接消掉了**。

剩下的就是：**人类选好回答的概率 = sigmoid(策略模型和参考模型在好回答与坏回答上的对数概率比之差)**。

**这整个表达式里，已经没有 Reward Model 的任何参数了**——全部都是策略模型和参考模型的概率。**所以你可以直接对这个式子做极大似然——Reward Model 就被数学上等价地消掉了**。

### 三个直接好处

DPO 的简化带来：

1. **砍掉了 Reward Model**
2. **砍掉了 Value Model**
3. **砍掉了 PPO 的采样循环**

**训练过程从四个模型变成了两个模型**：策略模型 + 参考模型。**工程复杂度直接降了一个量级，训练也更稳**。

## 反事实：DPO 没有问题吗？

**有，而且问题不小**——**面试官会追这一刀**。

### 核心问题：Offline 分布偏移

**最核心的问题是 DPO 是 offline 的**——**它只学习训练数据中已有的偏好**，**不会像 PPO 那样在线采样探索**。

打个比方：**DPO 就像你拿着一份去年的考试排名去指导今年的学习**：

- **如果你今年的水平跟去年差不多**——这个排名还有参考价值
- **但如果你已经进步了很多**——**去年那份排名的信息量就不够了**——你需要重新考一次试才能知道自己现在到底该往哪个方向使劲

换成技术语言：**偏好数据的分布和当前策略的输出分布之间会出现偏移**——**这个偏移越大，DPO 的优化方向就越不准**。

### 社区的衍生：沿着减法的方向继续

正因为这个问题，后来出现了一系列 DPO 变体——**整个社区都在沿着同一个方向做减法**：

| 变体 | 在做什么减法 |
|---|---|
| **Iterative DPO / Online DPO** | 训练过程中**不断用当前模型重新采样、重新标注偏好对**——缓解分布偏移 |
| **IPO** | 解决 DPO 在偏好标签过于确定时的过拟合问题 |
| **KTO** | 更激进——**连成对的偏好数据都不要了**，只需要"这个回答好"或"这个回答不好"的单条标签就能训练。**KTO 还去掉了参考模型**，用回答的平均对数概率本身作为隐式奖励 |
| **ORPO** | **更直接——把 SFT 和对齐合成了一步** |

整个社区在不断追问：**到底哪些组件是可以省掉的**？

## GRPO：换了一条路的减法

DeepSeek 最早在训练 **DeepSeek Math** 时提出了 GRPO，**后来在 R1 中大规模使用**。

### 出发点

它面对的问题是：**对于数学推理、代码这类任务，我能直接验证答案对不对——为什么还要人来标偏好**？

### 做法

GRPO 的具体做法：

1. **对同一个 prompt 一次性采样一组输出**（比如 16 个或 64 个）
2. **用规则或验证器直接打分**——数学题看最终答案对不对，代码就跑测试用例
3. **在这一组输出内部算每个输出的优势值**——**用组内均值作为 baseline，用组内标准差做归一化**

### 在强化学习谱系里的位置

GRPO 本质上是 **REINFORCE 算法的一个变体**：

- **经典的 REINFORCE** 用单个样本估计 → 梯度方差很大，训练不稳
- **加 baseline 来降方差** → 但 baseline 怎么估又是个问题
- **PPO 专门训了一个 value model 来干这件事** → 多一个模型的开销
- **GRPO**：**我不需要学一个 baseline——直接采一组样本，用这一组的平均奖励当 baseline 就行了**。**样本量够大的时候，这个估计已经足够准**——而你**省掉了整个 value model**

同时 GRPO 的梯度更新还是 **PPO 风格**：

- 保留了 **clip 机制** 来限制每一步的策略更新幅度
- 加上 **KL 散度约束** 不让策略偏离参考模型太远

**所以 GRPO 不是一个全新的东西——而是把 PPO 和 REINFORCE 的优点做了一个非常务实的组合**。

## 演进谱系总结：一条清晰的减法路径

| 范式 | Reward Model | Value Model | 在线采样 | 模型数 |
|---|---|---|---|---|
| **RLHF + PPO** | 需要 | 需要 | ✓ | **4** |
| **DPO** | 不需要 | 不需要 | ✗（offline）| **2** |
| **GRPO** | 不需要 | 不需要 | **✓** | **2** |

**GRPO 做到了"跟 DPO 一样轻量"的同时保留了"PPO 在线探索"的优势**。

## 反事实：GRPO 的适用边界

**但也要注意 GRPO 的适用边界**——**它高度依赖"可验证的奖励信号"**：

- **数学和代码天然有标准答案**——**所以 GRPO 效果极好**
- **但如果是开放式对话、创意写作这类没有明确对错的任务**——**你拿什么规则去打分**？
- 这时候你**还是需要回到"人类偏好"或者"学习出来的 Reward Model"**

DeepSeek 在 R1 的训练中确实是这么做的：**推理任务用 GRPO + 规则奖励，通用对齐阶段还是要用偏好数据**。

## 一条主线讲清楚的答题方式

面试时**不要把这三个东西当成三个独立的算法来背**——你要讲清楚一条主线。

**这条演进路径的本质，是不断在追问**：**对齐这件事，哪些环节是真正必要的？哪些是可以被简化掉的**？

- **RLHF** 证明了"用人类偏好做强化学习"这条路走得通——**但它太重了**
- **DPO** 通过一个精巧的数学变换发现"**Reward Model 其实可以被等价地消掉**"——代价是**失去了在线探索能力**
- **社区围绕 DPO 衍生出一大批变体**——不断尝试去掉参考模型、去掉成对数据、合并 SFT 阶段——**本质都是同一个方向的减法**
- **GRPO** 换了一条路——发现在**可验证任务**上，**连人类偏好标注本身都可以不要**——用规则奖励加组内对比就够了，同时用一个**巧妙的 group baseline 设计**把 value model 也省掉了

**每一步都是在做减法——但减的不是效果，而是不必要的复杂度**。

## 加分点：未来方向

**这条线大概率还没走到头**：

- **当前 GRPO 的限制**：奖励信号必须可验证
- **但如果未来出现足够强的通用自动评估器**（比如用一个很强的 LLM 当 judge）——**那类似 GRPO 的在线强化学习范式就有可能推广到所有任务上**
- **到那时候**——**人类标注可能真的只需要出现在最早期的种子数据里，后面的对齐过程可以完全自动化**

**这样面试官会觉得你不只是在复述论文，而是真的在跟踪和思考这个方向**。

## Open Questions

- **DPO 的数学推导**——"KL 约束的闭式解 → reward 等价于对数概率比 → BT 里常数项消掉"——**这个推导链每一步都需要稍紧严的数学训练**。如果有面试官追问"为什么 KL 约束的拉格朗日有闭式解"，**我能不能从测度论 / 变分原理的角度严谨推导**？这一步本节点没碰到。
- **KTO 完全去掉成对偏好数据 + 去掉参考模型** 的设计——它"用回答的平均对数概率作为隐式奖励"具体怎么避免训练崩溃？文章只点了名字，没展开机制。
- **GRPO 的"组内 baseline"** 在数学上是 **REINFORCE-with-baseline** 的某种特例吗？经典 baseline 理论里有"最优 baseline 是 advantage 的方差最小化"——**组内均值在什么条件下等于这个最优 baseline**？这条线有理论 sweet spot 但本节点没展开。
- **"通用自动评估器（LLM-as-judge）"** 替代人类偏好——但 judge 自己的 bias 会不会累积？这是 RLAIF 路线的一个开放问题。
