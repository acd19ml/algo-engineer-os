# SFT and RL: Capability Activation Relationship

## 它要解决什么问题

面试官问："为什么大模型训练要先 SFT 再强化学习？能不能跳过 SFT 直接上 RL？能不能只做 SFT，不做 RL？"

**这几个问题答到点子上，能直接体现你对训练范式的理解深度**。

要回答透彻，得先看清一个反直觉的事实——**预训练模型学到的能力，远比 SFT 后展现出来的多**。

## 朴素直觉为什么不够：预训练能力是冰山

预训练阶段，模型吃掉了整个互联网的数据——见过代码、论文、小说、百科、论坛帖子、各种语言的文本。**这些数据里蕴含了大量的知识和能力**。

**但这些能力是"沉睡的"**——模型不知道什么时候该调用什么能力。

SFT 的作用是：**用有限的示范数据告诉模型"遇到这种问题用这种方式回答"**。

但问题来了：

- **SFT 数据是人标的，数量有限、覆盖面有限**
- 你不可能穷举所有场景、所有问法、所有合理的回答方式
- **SFT 能激活的只是预训练能力的一个子集**——这个子集**由你的数据分布决定**

打个比方：**预训练模型像一个读过所有书的人，但从来没和人聊过天**。SFT 像是给他看了 100 段对话示范，告诉他"应该这样聊天"——他确实学会了这 100 种场景怎么应对。**但遇到第 101 种场景，他的策略是什么？是从这 100 个里面找一个最像的套上去用**。

**这就是 SFT 的天花板——泛化能力受限于训练数据的覆盖面**。

## 反事实：强化学习怎么突破这个天花板

强化学习不给模型看标准答案——而是**让模型自己生成回答，然后告诉它"这个回答得多少分"**。**模型要做的是"最大化这个分数"**。

这个机制带来一个关键变化：**模型开始探索训练数据之外的空间**。

它会尝试不同的回答策略——**有些是 SFT 数据里见过的，有些不是**。**只要能拿到高分，模型就会强化这种行为**。

这意味着：**模型可以调用预训练阶段学到、但 SFT 数据没覆盖到的能力**。

### 具体例子：数学题

假设你的 SFT 数据里数学题的解答都是 **直接给出计算过程和答案**——模型学会了这种模式。

**但如果遇到一道更难的题**，需要 **分情况讨论 / 先做辅助构造 / 验算**——这些策略 SFT 数据没教过，模型就不会主动用。

**强化学习阶段**——模型如果尝试了"分情况讨论"的策略，结果拿到了高分，**它就会学到"原来这种场景可以这么处理"**。

**这个能力本来就在预训练里学过**，**只是 SFT 没激活**——**强化学习把它挖出来了**。

这也解释了为什么强化学习后的模型经常展现出 SFT 数据里没有的行为模式——**不是凭空学会的，是被激活的**。

## 反事实：为什么不能跳过 SFT 直接上 RL

DeepSeek 有一个 **R1-zero** 模型——**没做 SFT 训练，直接用代码和数学题进行强化学习训练**。

为什么这种情况下能跳过？因为**代码和数学题有明确的奖励信号**——对就是对，错就是错。如果**奖励信号足够清晰、输出格式有约束、base 模型足够强**——SFT 不是必须的。

**但 R1-zero 还是有挺多问题**：

- 可读性较差
- **语言混合**（中英文混着输出）
- 答案是对的，**但根本看不懂**

所以最后真正能用的模型还是**用了 SFT 重新训练**。

### 一般情况下的根本困难

**强化学习依赖模型的探索**——如果起点太差，**模型探索出来的回答全是垃圾**。再加上**奖励信号没有区分度**——模型学不到任何有用的东西。

**SFT 的作用是把模型拉到一个合格的起点**——知道对话格式、会正常回答问题、输出基本可读。**在这个起点上做探索，才能探索出有意义的变体**——强化学习才有效果。

类比：**教一个人打乒乓球**

- **SFT** = 教他正确的握拍姿势、基本的发球接球
- **强化学习** = 让他打比赛——赢了加分、输了扣分

**如果跳过基础训练直接打比赛——他连球都接不住，比赛反馈没有任何意义**。

## 反事实：什么场景只做 SFT 也够用

如果**任务场景足够收敛、边界足够清晰、SFT 数据质量足够高、覆盖足够全**——SFT 就能把需要的能力激活完，**不需要额外探索**。

例：**特定格式的信息抽取任务**——输入输出都是固定模式。**SFT 已经能覆盖所有合理情况，强化学习没有多余的探索价值**。

但如果场景是 **开放式对话、复杂推理、创意写作**——用户输入和合理输出的空间都很大，**SFT 不可能穷举**——这时**强化学习的探索能力变得必要**。

## 完整答案：训练范式的三层认知

预训练模型的能力是**冰山**：

- **SFT 激活的是水面上可见的部分**——由你的数据分布决定边界
- **强化学习通过探索和奖励信号能激活水面下更大的部分**

**SFT 提供起点，强化学习扩展边界——两者是互补关系**。

| 操作 | 后果 |
|---|---|
| 跳过 SFT | 起点太差，探索无效 |
| 只做 SFT | 边界太窄，能力受限 |
| 先 SFT 后 RL | 起点合格 + 边界扩展 |

**这个答案的核心洞察**：**强化学习不是教模型"新能力"，而是激活模型已有但未被 SFT 覆盖的能力**。

## Open Questions

- DeepSeek R1-zero 的 "**无 SFT 直接 RL**" 路径——什么场景下这条路可行？文章列了"明确奖励信号 + 输出格式有约束 + 强 base 模型"三个条件，**但每条的具体阈值没说**。这条线在 reasoning 类任务上有大量探索，本节点没展开。
- **强化学习的奖励信号设计** 是核心难题——本节点只讲了"分类 / 数学题"这种**有明确正确答案**的场景。**开放式对话**的奖励信号怎么设计？RLHF / DPO / RLAIF 的对比这一层本节点没碰到。
- **SFT 数据"穷举"的下限**：如果 SFT 数据足够大、足够丰富——能不能完全替代 RL？这条线和 scaling laws 相关，本节点没展开。
