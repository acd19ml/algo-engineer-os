# SFT Training Strategy

## 它要解决什么问题

面试问 SFT 超参数怎么选时，"学习率小一点，epoch 两三轮，用 cosine 调度"只能算经验答案。真正的主线是：SFT 不是从零学习语言，而是在一个已经很强的模型上做精细行为调整。

所以所有训练策略都围绕同一个矛盾：**让模型学到新行为，同时不破坏旧能力**。学习率、epoch、batch size、调度、warm-up、数据混合，都只是这个矛盾的不同旋钮。

## 学习率：不是越小越安全

SFT 学习率通常比预训练低 1-2 个数量级。预训练常见在 `1e-4` 到 `3e-4`，SFT 常见在 `1e-5` 到 `5e-5`。

原因很直接：预训练从随机初始化开始，需要大幅调整参数；SFT 的起点已经具备语言和通用能力，只需要小幅改变行为模式。学习率太高，会把预训练能力冲掉，表现为灾难性遗忘。

但太低也不是安全。学习率过低时，模型在有限训练步内学不到目标行为，微调等于白跑。

一个实用搜索区间是从 `1e-5` 开始，试 `1e-5 / 2e-5 / 5e-5`。如果数据量大到几千条以上，学习率对结果会更敏感；如果只有几十到几百条，小数据场景里 batch size 往往比学习率更主导。

## Epoch：看总步数，不只看轮数

SFT 常见只跑 1-3 个 epoch，因为数据集通常不大，反复看同一批数据很容易从学习行为变成记忆样本。

但 epoch 本身不是绝对尺度。50 条数据跑 3 个 epoch，只看了 150 次样本，可能不够；1 万条数据跑 2 个 epoch，就是 2 万个样本暴露，通常已经足够。

更稳的判断是看总训练步数：`样本数 × epoch / batch size`。多数 SFT 任务在几百到几千步内比较合理。训练是否继续，不只看训练 loss，而要看验证 loss、输出多样性和通用能力是否开始退化。

反事实地看，如果只用固定 "3 epoch" 规则，小数据可能学不够，大数据可能过拟合。

## 调度和 warm-up：短训练里差异小，但启动必须稳

Cosine 调度前期下降慢，中期加速下降，后期平滑逼近 0。它适合让模型前期充分学习，后期温和收敛。

Linear 调度更简单，学习率匀速降到 0。在 SFT 这种训练步数不长的场景里，linear 和 cosine 经常差异不大。

Constant with warm-up 先升到目标学习率，然后保持不变。小数据场景有时更好，因为它不会在训练后期过早把学习率降得太低，让模型在有限步数内仍有学习信号。

无论用哪种调度，warm-up 基本都应该有，常见设置为总步数的 5%-10%。它防止训练初期梯度不稳，尤其在小 batch 下更重要。

## Batch size：小数据不要盲目放大

预训练里大 batch 常常更稳定，但 SFT 小数据不一定。数据多样性有限时，过大的 batch 会把梯度过度平滑，模型学不到细粒度行为。

经验范围是 4-32。几十到几百条数据，用 4-8；几千条以上，可以试 16-32。小 batch 会让 loss 曲线抖动，但这种抖动不一定坏，它可能保留了样本差异带来的学习信号。

所以小数据场景的优先级不是先疯狂调学习率，而是先找到合适 batch size。

## 过拟合和遗忘不能只看 loss

传统过拟合信号是训练 loss 降、验证 loss 升。SFT 里这个信号仍然有效，但不够。

第二类信号是输出多样性下降。模型开始对不同问题给出高度相似回复，或者频繁复用训练数据里的固定句式。这在 loss 上可能不明显，但实际使用时很明显。

第三类信号是通用能力退化。目标任务变好，代码、推理、通用问答却变差。这说明模型被窄分布数据拉偏。

应对顺序通常是：减少训练轮数，增加数据多样性，降低学习率，适度正则化，混入通用数据。

## 灾难性遗忘：工程上优先 LoRA + 数据混合

缓解遗忘有很多理论方法，比如 EWC 通过 Fisher 信息约束重要参数。但在大模型 SFT 里，EWC 开销高，实际工程里不常用。

更常见也更划算的是 LoRA 加数据混合。LoRA 只调整小部分低秩参数，天然限制模型变化幅度；数据混合则在 SFT 数据里加入 10%-30% 通用指令数据，为旧能力保底。

这不是浪费训练预算，而是在防止模型把所有能力都挤向一个窄任务分布。

## 课程学习：有条件有用

如果数据难度跨度很大，比如既有简单格式转换，也有复杂多步推理，先学简单再学困难可能让训练更稳。

但课程学习需要定义难度：按回复长度、困惑度、人工等级，都会有偏差。更实用的折中是前期提高简单样本采样权重，后期逐渐增加困难样本，而不是严格排序所有数据。

如果数据难度本来均匀，课程学习收益可能很小，反而增加复杂度。

## Open Questions

- SFT 的最优 batch size 与数据多样性之间的关系需要更可观测的诊断方法。只看 loss 抖动很难判断是噪声还是有效学习信号。
- 课程学习在 SFT、RLVR、GRPO 中的作用可能不同。SFT 里按难度排序是否会牺牲分布随机性，需要实验确认。
