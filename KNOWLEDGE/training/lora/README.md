# LoRA: Low-Rank Adaptation

## 它要解决什么问题

面试官问 "LoRA 的原理是什么？"

如果你上来回答 "LoRA 就是冻结原始权重，然后加一个低秩分解的旁路，训练时只更新这个旁路的参数让模型学会新的能力"——**对，但面试官早就听腻了**。

面试官真正想考的是 **两个核心问题**：

1. **为什么把一个小矩阵加到原权重上就能注入新的能力**？
2. **为什么这个小矩阵可以是低秩的**？

要答透，得分三层讲——基础原理、为什么有效、为什么低秩成立。最后再加上**显存估算**这一层实操能力。

## 朴素直觉为什么不够：先把基础讲准

全参数微调的时候，更新的是整个权重矩阵——**算力和显存的资源消耗特别大**。

LoRA 的做法：把维度为 `D × D` 的权重的更新量，**分解成两个小矩阵的乘积** `A` 和 `B`：

- `B` 矩阵维度 `D × R`
- `A` 矩阵维度 `R × D`
- `R` 远远小于 `D`，**但相乘后维度仍然是 `D × D`**

训练的时候**原始权重 W 完全冻结，只更新 A 和 B**——资源消耗远小于全参数微调。

### 重要的初始化细节

- **B 矩阵被初始化为全 0**——意味着训练刚开始时更新权重等于 0
- 模型的行为和预训练时完全一致
- **新的修正是在训练过程中逐渐学出来的**

推理时把 `B × A` 的结果加回原来的权重矩阵——**不增加任何推理开销**。

## 反事实：为什么"加法"就能注入新能力

这是最关键的一层。

### 权重矩阵是一个映射函数

要理解 LoRA，先要理解：**神经网络里的每一个权重矩阵，本质上是一个映射函数**——输入一个向量，经过矩阵输出另一个向量。**这个向量的方向和大小决定了这一层往下游传递什么信息**。

可以把矩阵想象成一个**棱镜**——输入信息是光，打进来之后棱镜把它折射到特定的方向。下一层再接住这束光继续处理。

### LoRA 的修正在做什么

加了 LoRA 之后，推理时的计算变成：

```
(W + ΔW) × x = Wx + ΔW × x
```

- **前半部分 `Wx`** 是原始模型的输出
- **后半部分** 是 LoRA 旁路产生的一个**修正向量**

这个修正向量具体在干什么？

- **A 矩阵**：先把输入 x **压缩到一个 R 维的低维空间**——相当于在问"这个输入在任务相关的几个关键特征上的响应是什么？"
- **B 矩阵**：把这 R 个特征的响应**重新映射回原来的高维空间**——生成一个修正方向

**这个修正方向叠加到原来的输出上，就形成了一个新的输出方向**。

### 关键的几何直觉：只在 R 个方向上修正

`ΔW` 的秩是 R——**意味着它只能在高维输出空间中的 R 个方向上产生修正**。

假设输出空间是 4096 维、R = 8——**那么 LoRA 只修改了其中 8 个方向**。**剩下 4088 个方向上的输出和原来完全一样**。

**这 8 个方向不是随便固定的坐标轴，而是在训练过程中学出来的**——**模型自己决定该往哪几个方向上做调整**。

**所以 LoRA 的修正是"高度定向"的**——不是漫无目的地改动参数，**而是在训练过程中学习到几个最关键的方向，然后精准地在这几个方向上做调整**。

### 反事实：只改 8 个方向能有多大影响

这就涉及到 **Transformer 的层叠结构**了。

**每一层的输出是下一层的输入**。你在某一层的 8 个方向上做了偏移，**这个偏移传到下一层之后，经过非线性激活和注意力计算，它的影响会扩散到更多的方向上**。

**经过几十层的传递，最初几个方向上的小偏移可以显著改变模型最终的输出行为**。

**就像一条河在上游被轻轻推了一下，经过层层弯道之后，到下游已经走向了不同的方向**。

### 为什么 LoRA 比全参微调抗遗忘

因为**每一层的 LoRA 都只修改少数几个方向，绝大部分的表征空间没有被干扰**——**这就是为什么 LoRA 微调后的模型相比全参微调更不容易出现灾难性遗忘**。

注意——**原因不是"原始权重没被动过"这么简单**。**推理时用的就是 W + ΔW，权重确实变了**。**真正的原因是低秩约束限制了修改的范围**——让变化只集中在一个低维子空间里，**大部分原始能力自然就保留了**。

工程上还有个好处：**你随时可以把 ΔW 摘掉，100% 恢复原始模型**——这是全参微调做不到的。

## 反事实：为什么 ΔW 可以是低秩的

理解了"加法为什么能注入新能力"之后，面试官会追问第二个问题——**"你只用秩 8 的矩阵就能达到接近全参微调的效果，凭什么？"**

答案在于一个叫 **Intrinsic Dimensionality（内在维度）** 的概念。

### 核心论点

虽然大模型的参数空间有几十亿维，**但在微调的时候，真正起作用的参数变化方向只集中在一个非常低维的子空间里**。

为什么会这样？

- **预训练阶段，模型已经在海量数据上学到了非常好的通用表征**
- 当你微调去适配一个具体任务时，**不需要重建整个表征空间**
- **只需要在几个关键方向上做调整就够了**

类比：**你已经有了一张画得很好的世界地图**，现在只需要在某个区域加几个标注，**而不是重新画一整张地图**。

### 实证支持

2020 年有一篇论文专门验证了这个现象——**很多下游任务微调的内在维度只有几百，甚至几十**。

也就是说：**全参微调时，绝大部分参数更新是冗余的**——**真正有效的更新集中在一个极低维的子空间里**。

LoRA 就是在利用这个性质：

- **A 矩阵** 负责找到那几个关键方向
- **B 矩阵** 负责把这些方向上的调整映射回完整的参数空间
- **设置 R 就意味着 "你认为任务适配只需要 R 个方向就够了"**

**所以低秩分解不是"无奈的妥协"——它之所以有效恰恰是因为微调这件事本身就具有低秩的数学性质**。

## 实操层：rank 怎么选 + LoRA 加在哪些层

### Rank 选择

很多人觉得 rank 越大越好——**实际情况是 rank 过大时**：

- 引入了太多可训练参数
- **数据量不够大的情况下反而容易过拟合**
- **rank 超过一定阈值之后效果提升非常小**——因为已经覆盖了内在维度的主要方向，多出来的维度捕获的都是噪声

实际选择：

| 任务复杂度 | rank |
|---|---|
| 简单（情感分类、意图识别） | 4-8 |
| 中等（多领域指令微调） | 16 |
| 复杂（全新输出格式学习） | 32 |
| **超过 64** | **基本不会有明显收益**——如果 64 还不够，**说明你的任务可能已经不适合 LoRA 了，该考虑全参微调** |

### LoRA 加在哪些层

- **原始论文的结论**：加在注意力层的 **Q 和 V 投影**上效果最好
- **后来大量工程实践发现**：把 LoRA 加在**所有线性层**上（K 投影、输出投影、FFN 里的上下投影），整体效果**往往更好**
- **代价**：可训练参数量多几倍，**需要根据显存预算取舍**

## 显存估算：四部分拆解

启动一个 LoRA 微调任务，**怎么在跑之前就估算需要多少显存**？

把显存拆成四部分，**逐个估算**：

### 1. 模型参数

LoRA 的核心是冻结原始模型权重——**只训练新增的低秩适配器**。

- 原始模型参数通常用半精度加载，**每个参数占 2 字节**
- 一个 7B 模型光加载权重就要 **14GB**
- LoRA 新增的可训练参数量很少——**通常只有原模型的 0.x%**——**这部分可以忽略**

### 2. 优化器状态

如果用 AdamW，每个**可训练参数**需要额外存 12 字节：

- 4 字节一阶动量
- 4 字节二阶动量
- 4 字节参数主副本

**因为 LoRA 只训练少量参数，这部分开销被大幅压缩**：

- 7B 模型的 LoRA 参数大概**几千万**
- 优化器状态只占**几百 MB**

### 3. 梯度

**只需要为可训练参数存储**——用半精度的话，几千万参数也就**几十 MB，可以忽略**。

### 4. 激活值（最难算的部分）

激活值是前向传播过程中**每一层的中间结果**——需要保存供反向传播用。

大小取决于：**Batch Size × 序列长度 × 隐藏层维度 × 层数**。

粗略公式：

```
激活值 ≈ batch_size × seq_len × hidden_dim × num_layers × 2 字节 × 系数
```

**系数一般在 10-30 之间**——取决于具体架构。**激活值通常是显存的大头**。

**好消息**：可以开 **梯度检查点**（gradient checkpointing）——**用时间换空间**：

- 激活显存能降到**原来的 1/3 到 1/5**
- 代价是训练时间增加 20-30%

### 具体例子

**7B 模型 LoRA 微调（batch=1, seq_len=2048）**：

| 部分 | 显存 |
|---|---|
| 模型参数（半精度） | 14 GB |
| LoRA 优化器状态（35M × 12 字节） | 约 400 MB |
| 梯度（35M × 2 字节） | 约 70 MB |
| 激活值（开梯度检查点） | 2-4 GB |
| **总计** | **16-20 GB** |

**一张 24GB 的 4090 能跑，还有余量稍微加大 batch size**。

### 实操技巧

**激活值很难精确算**——**最靠谱的方法**：

1. 先用 batch_size=1 跑一次，看实际显存占用
2. **线性外推**——激活值和 batch_size 基本成正比

如果 batch=1 测出来 18GB，那 batch=2 大概就是 18 + (2-4)GB。

### 经验法则

**LoRA 微调的显存通常是全参数微调的 1/3 到 1/4**——**主要省在优化器状态和梯度上**。

## 总结：四步答题框架

1. **基础原理**：冻结 W，加 BA 旁路，B 初始化为 0
2. **加法为什么有效**：每个权重矩阵是映射函数；LoRA 的修正向量在少数几个方向上改变每一层的输出；这种改变通过层叠结构逐层传递放大；同时低秩约束保证大部分原始表征空间不受干扰，**抗遗忘能力远强于全参微调**
3. **低秩为什么成立**：微调的内在维度很低；预训练已经学好了通用表征；任务适配只需要在低维子空间里调整
4. **实操经验**：rank 怎么选、加在哪些层、什么场景用 LoRA / 什么场景该上全参微调、显存估算四部分拆解

**到第 4 步，面试官就知道你不是背论文来的——而是真正理解了 LoRA 的设计逻辑，并且在实际项目里用过、思考过**。

## Open Questions

- **B 初始化为 0** 是为了让训练初期模型行为不变——**但 A 是高斯初始化的**。如果 A 也初始化为 0 会怎样？这个 ablation 没看到。
- "**LoRA 加在所有线性层比只加 QV 效果更好**" 是经验观察——**有没有理论解释为什么 FFN 层的 LoRA 也很有效**？这条线和 transformer 各层的功能分化相关。
- **rank=64 之后收益消失** 是经验阈值——**这个阈值和模型大小、任务复杂度的依赖关系**怎样？没看到系统化的研究。
- **LoRA 多任务合并**：训完多个 LoRA adapter 后，能不能直接相加？数学上 ΔW1 + ΔW2 是否等价于 "同时学两个任务"？这条线和 model merging 相关，本节点没展开。
