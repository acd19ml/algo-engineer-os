# Q: 讲一次你在极限时间内把局面救回来的经历

> 也适用于："讲一次你顶住压力的经历"、"讲一次你 ownership 体现"、"讲一次你超出预期交付"

## STAR

### Situation

七牛云 ZeroOps 项目的某次 milestone 路演前夜。这是第一次路演失败 + CEO 拍板 MVP 形态 + 人员动荡的连环背景下进行的下一次路演（详见 `qiniu-ceo-pivot-decision.md` 和 `qiniu-team-turbulence-handoff.md`）。

**临近路演的现状**：

- **产品联调进度不达预期**——Agent 模块、Mock 系统、前端展示三方串联还有 bug
- **演示流程不完整**——MVP 形态明确了但展示路径还没走通端到端
- **PPT 没准备好**——技术内容很多但还没整理成"让导师听得懂"的演示版本
- 团队整体处在"路演会再次失败"的隐忧中

第一次路演失败的导师批评仍历历在目——**"汇报最重要的是让观众听懂"、"应该展示产品整体流程而不是技术细节"、"演示文档准备不足"**。

### Task

我作为组长 + Agent 设计 + 后端联调，需要在路演前**不让历史重演**。具体要做到三件事：

1. 让端到端演示路径跑通（Agent 模块 → Mock 系统 → 前端展示）
2. 让 PPT 能让非技术听众（其他业务线的工程师 / 导师）听懂
3. 兜底方案——即使现场 demo 出问题也能展示成果

### Action

#### 加班补救三件事

**1. 加班把端到端联调跑通**

- 优先级排序——只保证 demo 路径上需要的接口跑通，其它非关键路径接受 Mock 数据兜底
- 接口对齐冲刺——前后端 + Agent 模块 + Mock 系统四方对齐 demo 路径上的所有接口
- 数据流验证——从告警触发 → Agent 根因分析 → 修复建议输出 → 前端展示，整条路径手动跑通几次确保稳定

**2. 重做 PPT 让听众听得懂**

- **从"我们做了什么"换成"我们解决什么问题 + MVP 边界 + 核心 demo"**——前几页直接说核心价值，技术细节延后
- 加入产品流程图（不是技术架构图）——让非技术听众也能看懂"用户怎么用、系统怎么反应"
- 删减专业名词或加注解——避免第一次路演那种"听众云里雾里"的状况

**3. 准备 demo 录屏作为兜底**

- 把端到端 demo 走过的最稳定一次录屏下来
- 现场 demo 万一出问题就放录屏，确保至少能看到产品形态而不是技术故障

### Result

- **最终路演呈现超出所有人预期**——包括团队自己、其它组、导师们
- 这是项目最关键的一次"翻盘"——从第一次路演的失败 + 团队信心不足，到这次的超预期，把项目从"可能失败"拉回到"被认可"
- **项目最终获七牛云第三届 1024 创作节优胜奖**
- 路演成功还有一个隐性收益——团队信心恢复，后续推进虽然仍有挑战但没有再次"卡死"

## 可强调的能力信号

- **极限时间 ownership**——deadline 前一晚不是抱怨而是主动加班把进度补上来
- **优先级判断**——区分"demo 路径上的必要事项" vs "非关键路径可接受 Mock"，**用 80/20 思路把时间花在对路演成败影响最大的事**
- **听众导向**——从第一次路演失败学到"PPT 要为听众设计"，主动拉低技术细节、抬高产品故事
- **兜底意识**——准备 demo 录屏作为现场失败的 fallback，**ship 工程师的本能**
- **从失败学习**——把第一次路演的导师批评直接转化为这次的具体改进动作

## 最常见的深挖问题（提前准备）

1. **"具体加班加到几点 / 多长时间"** → 答：路演前一晚（具体到几点能补充则补充），如果你不记得了诚实说"几个晚上的连续加班"
2. **"为什么团队其他人不一起加班"** → 答：(a) 我作为组长接的本来就是协调 + 联调的位置；(b) 团队信心不足时让组员都通宵反而可能让局面更糟；(c) 取舍：组长承担额外压力换团队保持节奏
3. **"PPT 改了哪些具体的东西"** → 答：(a) 前三页换成"问题 + MVP 边界 + 核心 demo"；(b) 删了多少多少专业名词；(c) 加入产品流程图替代技术架构图
4. **"demo 现场真的稳定吗"** → 答：稳定是因为我们准备了**真实 demo + 录屏 fallback**两套方案，且 demo 路径上的接口路径都验证过多次
5. **"这次路演后项目最终怎么样"** → 答：**项目获七牛云第三届 1024 创作节优胜奖**。但 MVP 仍停在 Mock 阶段未上线生产，Agent 模块独立跑通但端到端主链路未完整接入。**路演成功 ≠ 上线成功**，这是我从这件事学到的额外一层
